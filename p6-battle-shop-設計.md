# P6設計書: クイズバトル＋ショップ

作成: 2026-07-05 ／ 親: feature-spec.md §8 ／ 状態: 骨子案（ユーザー承認前）
方針の土台: バトル=正解で攻撃のターン制(A案)／通貨=学習で貯まるコイン／
実在IP不使用・課金なし・射幸性なし（憲章遵守）

---

## 1. ねらい（教育との接続）

「学習する → コインが貯まる → アイテムを買う → バトルが有利 → もっと学習」の循環を作る。
バトルの出題は既存クイズ204問を使うため、**バトル自体がクイズ学習の反復**になる。
コインは学習行動からのみ得られる（時間経過・ログインボーナスでは配らない）。
復習(再プレイ)でもコインが出るのは意図的——「もう一度解く」動機として機能させる。

## 2. バトルの基本ルール（シンプル・ハート制）

- プレイヤー(相棒モンスター) vs 敵モンスター(CPU)のターン制。
- **正解 → 相棒が技を出し、敵に1ダメージ**。**不正解 → 敵が攻撃し、ハートが1減る**。
- プレイヤーは **ハート3つ**。敵のHPは難易度で変える:
  - やさしい: 3（=3問正解で勝ち）
  - ふつう: 4
  - むずかしい: 5
- ハートが0になったら負け。ただし**負けても罰なし**（励ましメッセージ＋
  「アイテムを使ってみよう」「やさしいで練習しよう」の提案。失敗を罰しない原則）。
- 出題は既存クイズ204問から、選んだ難易度のプールでランダム（直近に出た問題は避ける）。
- 相棒のレベルが高いと、たまに「かいしんのいちげき」（2ダメージ）が出る
  （育成との接続。確率はLvに応じて少しずつ。例: Lv1=5% → Lv12=25%）。

## 3. 敵の構成（オリジナルモンスター9体）

各難易度に3体ずつ、順に解放（1体倒すと次が開く）。全て倒すとその難易度クリア。

| 難易度 | 敵の想定（絵はChatGPT生成・3Dルック） |
|---|---|
| やさしい | 小さくて可愛い系3体（例: ぷにぷにスライム・きのこ・ことり） |
| ふつう | 中ボス感3体（例: いたずらオバケ・カニロボ・とげとげ） |
| むずかしい | 強そう3体（例: ドラゴンの子・ゴーレム・ラスボス風） |

- 敵に属性・相性は**入れない**（低学年に概念過多。将来の拡張余地としてのみ温存）。
- 倒した敵は記録され、ずかんの「たおした あいて」コレクションに追加
  （相棒ずかん・バッジと同じ収集演出。9体コンプが目標）。

## 4. コイン（通貨）

### 入手（学習行動のみ）
| 行動 | コイン |
|---|---|
| パズルで新しく★を獲得 | ★1つにつき 2枚 |
| クイズ1問正解（セッション終了時に精算） | 1問 1枚 |
| タイピング クリア（自己ベスト更新時はボーナス+5） | 3枚 |
| おえかき 作品保存（1日の上限あり・乱発防止） | 2枚 |
| バトル勝利 | やさしい5 / ふつう8 / むずかしい12枚 |

- 過去の実績にも初回換算で付与（P6導入時、既存の★・記録から一括計算して
  「いままでの がんばり」として最初にまとめて渡す。導入日から急に貯まり始める
  より、これまで頑張った子が報われる形にする）。

### 表示
- ヘッダーに 🪙コイン数 を常時表示（⭐の隣）。

## 5. ショップ

ホームから入る「おみせ」。買えるものは2系統。

### A) バトルで役立つアイテム（消耗品・持てる数に上限）
| アイテム | 効果 | 価格(案) |
|---|---|---|
| 🧃 かいふくドリンク | ハートを1つ回復（バトル中に使用） | 10枚 |
| 👓 ヒントメガネ | 選択肢を1つ消す（**答えを考える助け**＝教育的） | 15枚 |
| 💪 パワーバンド | 次の正解のダメージが2倍 | 15枚 |
| 🛡️ まもりのたて | 次の不正解のダメージを防ぐ | 15枚 |

- 各アイテムは同時に3個まで所持（買い込み無双の防止）。
- バトル中、自分のターンの前に1個だけ使える。
- **強すぎ防止**: アイテムだけでは勝てない（正解しないとダメージが出ない）設計。
  アイテムは「間違えても立て直せる保険」の位置づけ。

### B) コレクション（見た目・非消耗）
| 種類 | 内容 | 価格帯(案) |
|---|---|---|
| 🎩 きせかえ | 主人公（探検家）の着せ替え8種・6スロット（head/face/neck/chest/waist/back）。詳細は replacement_instructions_for_code.md 参照 | 種類による |
| 🖼️ バトルのぶたい | ジャングルのぶたい／だいちのぶたい（2種・既存の難易度別3枚=草原/夕日/夜とは別枠・そちらは変更なし） | 50枚 |

- 買った着せ替えは主人公の表示（ヘッダー・バトル等）に反映。集める楽しさ担当（射幸性なし・
  ガチャにしない。**全品が定価で確実に買える**）。
  ※2026-07-07改訂: きせかえの対象を相棒→主人公に変更（source_docs_patch.md 適用）。

## 6. 画面の流れ

- ホームに「⚔️ クイズバトル」モードカードを追加（アイコンはChatGPT生成）。
- ホームに「🪙 おみせ」入口を追加（モードカードより小さめでよい）。
- バトル: 難易度選択 → 敵選択（解放済みから）→ バトル画面
  （上に敵・下に相棒、ハートと敵HP表示、クイズが出る、アイテムボタン）→
  勝敗演出 → 報酬（XP＋コイン）→ 敵ずかん更新。
- 記録: 保護者画面のログに「バトル」を合流（クイズ学習の一形態として集計）。

## 8. データ構造（保存）

- battle: { defeated: [敵ID], best: 難易度別勝利数 }（既存の予約枠 battle:{} を使用）
- coins: 数値（プロファイル別）
- items: { アイテムID: 個数 }
- cosmetics: { owned: [ID], equipped: ID }
- 書き出し/読み込みで完全復元（P5と同じラウンドトリップ試験を通す）。

## 9. 生成する絵（ChatGPT・プロンプトはChatが用意）

1. 敵モンスター9体（3Dルック・相棒と同じ画風・白背景）
2. モードアイコン「クイズバトル」1枚（既存4アイコンと同画風）
3. ショップアイコン or 店主キャラ 1枚
4. アイテム4種＋きせかえ＋背景2種は、まず**絵文字/自作SVGで開始**し、
   遊んで物足りなければ生成画像に差し替え（一度に全部生成すると量が多いため。
   敵9体とアイコンを優先）。
   ※2026-07-07改訂: きせかえは「相棒の飾り」→「主人公（探検家）の着せ替え」に
   読み替える（生成画像も用意済み・source_docs_patch.md 適用）。

## 10. やらないこと

- 通信対戦・ランキング（憲章: 通信対戦は作らない）
- ガチャ・ランダム報酬（射幸性なし。全品定価）
- 属性・相性システム（低学年に概念過多。将来の拡張余地）
- コインの時間配布・ログインボーナス（学習行動からのみ）

## 11. 実装の停止ポイント（Code向け・実装指示書に落とす際）

1. バトルの土台（ハート制・出題接続・勝敗・XP）ができた時点で一度停止→確認
2. コイン＋ショップ＋アイテムを実装→確認
3. 敵の絵・アイコンの差し込み→全体通し→デプロイ
