# P2追補2: 難易度別マップ背景（昼・夕方・夜）

作成: 2026-07-04 ／ 親: progland-実装指示書.md ／ 対象: Puzzle.jsx の島マップ

## 目的
難易度ごとにマップ背景を変える（同じ絵の使い回しを避ける）。
同一島の時間帯違いにすることで、確定済みの6拠点座標(ISLAND_POS)を3枚で共有できる。

## 素材（3枚。すべて1600×900・WebP・軽量化済み）
- `worldmap.webp`（既存・昼）… **やさしい**
- `worldmap-sunset.webp`（夕方）… **ふつう**
- `worldmap-night.webp`（夜・道と空き地はランタンで点灯）… **むずかしい**

sunset と night を `src/assets/` に追加する。

## 実装
- 難易度state（easy / normal / hard）に応じて背景画像を切り替えるだけ。
  例: `const MAP_BG = { easy: worldmapDay, normal: worldmapSunset, hard: worldmapNight }`
- **拠点座標 ISLAND_POS は3枚共通でそのまま使う**（構図一致を検証済み）。座標変更不要。
- 拠点ボタン・道・ラベルのオーバーレイは既存のものを流用。背景だけ差し替わる構造にする。
- 夜背景はやや暗いので、拠点ボタンや島名ラベルの視認性を軽く確認（必要なら
  ラベルの下地の不透明度を少し上げる程度。凝った出し分けは不要）。

## 検証観点
- 3難易度を切り替えると背景が 昼→夕方→夜 に変わり、6拠点が全マップで空き地に乗る。
- タブレット実機（iPad Safari 横・縦）で3枚とも拠点位置が破綻しない。
- 追加画像でビルドサイズが極端に増えないこと（各250KB前後なので問題ない想定）。
- `npm run verify` は従来どおり全162面PASS（背景差し替えはステージデータに無関係）。

## バージョン
完了で v2.0-P2.3 に上げてデプロイ。

## 備考
3枚は生成AIで「同一構図・ライティングのみ変更」で作成し、昼版の6座標を重ねて
全点が空き地中心に乗ることを画像処理で照合済み。将来さらに背景を足す場合も、
同じ構図を保てば座標を共有できる（構図が動いたらその背景だけ座標補正が必要）。
