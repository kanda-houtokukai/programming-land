### 決定事項（要点。詳細は指示書の正本を参照）

- [DECISION] Vite + React、**単一HTMLファイル出力**（vite-plugin-singlefile）。ダブルクリックで起動。ネット接続・サーバ・アカウント・課金なし
- [DECISION] 保存は localStorage ラッパー＋保護者画面に「きろくの かきだし／よみこみ」（JSON書き出し・復元）を必須実装（file:// でlocalStorageが消える場合の保険）
- [DECISION] ステージ・クイズ・タイピング課題は全てデータファイル分離。ハードコード禁止
- [DECISION] 難易度3段階（やさしい／ふつう／むずかしい）を全モード共通。記録は難易度別、解放条件は難易度内で完結
- [DECISION] フェーズは P0移行 → P1パズル150面以上 → P2クイズ200問以上 → P3タイピング → P4記録・バッジ。各フェーズの完了条件は指示書 §3
- [DECISION] P1ステージは手作業で作らず、ジェネレータ＋ソルバー自動検証（`npm run verify` 全数PASS）で量産
- [DECISION] やらないこと: オンライン機能・広告・外部連携、v1のUIトーン刷新、実在キャラ素材（絵文字と自作SVGのみ）

- ★ dev サーバ稼働中に `npm run build` しない（ビルド破損の既知事故と同型）
- ★ AudioContext はユーザー操作後に初期化（v1踏襲）
- ★ 保存データはバージョン番号つき、`{...newSave(), ...parsed}` のデフォルト値マージで読む（v1踏襲）
- ★ プレビュー成功≠完成。各フェーズ完了時にビルド→実機（Safari含む）確認
- 中間報告ポイント: P0完了時・P1ジェネレータ＋ソルバー完成時（量産前）・各フェーズ完了時に停止して報告
