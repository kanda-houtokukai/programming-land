## 4. 運用原則（この開発で確立）

- 1機能＝1フレッシュChat。状態はチャットでなくリポジトリに置く（新Chat/Codeは台帳→roadmap→feature-spec→brushup を読んで続く）。
- 役割分担: Chat＝設計・検証・アセット加工・指示書作成／Code＝実装・git・デプロイ・台帳記録／神田さん＝設計確認・リレー・実機確認。
- 裏取りは SHA固定 raw（push直後の raw-at-main は数分CDN遅延・叩きすぎで429）。
- 実装指示は「deploy＝実機確認のため／神田さんOKまで確定にしない」。体感値は「初期値→実機で微調整（定数1箇所集約）」。
- 絵文字不採用（アイコンは専用イラスト。既存バッジ26個の宣言形式など、宣言済みの例外を除く）。
- バトル演出のロジック境界（命中解決・突進・メッセージパネル・`pl-*`・結果シーケンス）は壊さない（新規は別名）。
- 環境判定は CSS @media で。JSの `matchMedia`／`ResizeObserver` はこの環境で誤発火・不発火の実績あり（b4b/b4l）。
- 大きい変更（スキーマ移行等）は Fable 5 でまとめ、roundtrip で移行を必ず確認。
- `npm run verify` は6本チェーン（verify-studio / test-studio-engine / test-roundtrip 込み）＝deploy のゲート。
