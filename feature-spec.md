## 2. データ構造（保存の骨格）

localStorage にプロファイル単位で保存。SCHEMA_VERSION を持たせ、項目追加/構造変更時はデフォルト値マージ＋移行関数で後方互換… 済（現行 SCHEMA_VERSION=3）。

保存に含む要素（実装済み）:
- profile: 名前・character（"boy"|"girl"）・作成日・dressup（{head,face,neck,chest,waist,back}・未装備 null）・**dex**（出会った/到達した相棒の姿を `${typeId}-${stage}` で記録＝図鑑の到達段階解禁に使用）。
- **partner（相棒・2026-07 刷新）**: `{ active:<typeId>, owned:[{ id, level, xp }…], egg: null | { xp } }`
  - **レベルは相棒ごと**（各 owned が個別の level/xp）。active＝部屋/バトルに出す1体。
  - egg＝保留中のたまごと孵化ゲージ進捗（詳細は§7）。
  - 移行: 旧「共有レベル」セーブ（〜b4i）は各 owned に当時のレベルを引き継ぎ（退行なし）。さらに旧 `{species, level}` も吸収。
- ずかん: 相棒15姿（到達段階ごと・上記 dex）＋倒した敵＋バッジ。
- puzzle / quiz / typing / art / badges / coins / items / cosmetics（{owned, equipped:{bg}}）/ battle（討伐記録・towerBest 等）/ powers / log / settings。

書き出し/読み込み: プロファイル単位でJSON・完全復元… 済（ラウンドトリップ試験・相棒の新スキーマ移行4ケース含め確認）。

## 4. かんがえるクイズ … 済

- カテゴリ: じゅんばん／きまり発見／なかまわけ／ロボット命令／ずのよみとり。3難易度・各問に解説・5問出題。

- **なかまわけに「〇〇の なかま」（軸名称形式＝属性フィルタリング）を追加 … 済**: 既存の仲間外れ探し（差を見つける操作）と同じ素材・同じ軸で、与えられた基準を当てはめる操作の問題。「名指しした軸に当てはまる選択肢がちょうど1つ」を機械検証。
- **なかまわけの軸を拡張（b4u）… 済**: やさ=カテゴリ＋**いろ・かたち**（あかい/きいろい/みどりいろの/まるい/ほそながい/しかくい・色や形が曖昧な絵文字は登場させない機械検証つき）／ふつう=はたらき9軸（+ひとを のせて はこぶ・あしが4ほん・みに つける・つちの なかで そだつ）／むず=抽象4軸（+ひとが つくった）＝軸の偏り（「たべられる」集中）を緩和。各難易度 仲間外れ24＋軸名称20で同じプールに混在（比率は初期値）。
- **1ゲーム内の重複防止（b4u）… 済**: 5問の選抜時に「実質同一問題（文言＋正解が一致）」「選択肢の丸かぶり」「じゅんばんの同じ話」「なかまわけの似た組（アイテム2つ以上共有）」「軸名称形式の同じ軸」を同席させない（R1〜R5・1ゲーム内だけの制約・セーブ記録は使わない）。

## 7. 育成（相棒モンスター）… 済（2026-07 全面刷新）

- **5タイプ×3進化＝15体**（もり／みず／ほのお／でんき／いわ）。自作イラスト（512px透過PNG・部屋/バトル/図鑑/ロアで共用）。
  - **デザイン方針**: タイプは葉/水などの記号を貼らず、**体の造形・素材・色で有機的に**表現。段階が上がるほど作り込み・威厳が増し、**最終進化は複雑・いかにも強そう/洗練**（例: もり＝古木の守護獣モリガルド／みず＝水竜ウズリュウ／ほのお＝炎の霊獣エンブレオ／でんき＝雷獣ライオウ／いわ＝大地の守護獣ガイオン）。CSS描画から `<img>` に移行。
- **はじめに5体から1体を選択**。残り4体は**たまごサイクル**で収集（下記）。
- **レベルは相棒ごと**。XP: パズル・クイズ・おえかき・タイピング・バトル勝利で**アクティブ相棒**に加算。
- **進化**: Lv5（stage2）・Lv12（stage3=最終）… 不変。進化で**サイズが大きくなる**（部屋・バトル共通・`partnerStageScale {1:1.0, 2:1.2, 3:1.45}`・スマホは比例縮小）。
- **たまごサイクル … 済**:
  - アクティブ相棒が**最終進化（stage3=Lv12）に到達した瞬間、たまご1個を即付与**（未所持タイプが残っていて・卵が保留中でないとき）。**卵は常に1個だけ**（溜まらない）。
  - 付与後、アクティブ相棒が得るEXPで**孵化ゲージ**が進む（`EGG_HATCH_XP=40`＝やさしいバトル3〜4回ぶん）。満了で**未所持タイプからランダムに1体（Lv1）**が仲間に。
  - それを育てて最終進化 → また即たまご → 孵化 → …（4回で5体コンプ）。＝1体ずつじっくり育てる反復。図鑑コンプは容易。
  - おうちの卵欄（b4s 改修済）: 看板は「たまご」（他の家具ラベルと同じRPG看板調）・egg.png＋**孵化ゲージ（巣の名札風・世界観色）＋状態文はゲージ直下**（「たまごを あたためてる」→「あと ちょっと」→「もうすぐ！」・白＋縁取りで床の上でも読める）。**卵をタップすると たまごモーダル**（卵の絵大きめ＋ゲージ＋状態文＋「あそぶと あたたまるよ」＋とじる）。孵化は従来どおりEXPで自動（選択なし）・孵化時は自動きりかえせず「おうちで きりかえられるよ」と案内。
- **ロア … 済**: 相棒タップで**架空の設定（ひらがな・数文）**を表示（なまえ/タイプ/すがた＋物語）。図鑑エントリからも。
- **図鑑**: 15姿。**到達した段階ごとに解禁**（profile.dex の `${id}-${stage}`・未到達は？シルエット・タイプ見出しは所持で表示）。集めて育てるほど埋まる。
- **相棒に着せ替え（飾り）は付けない**方針は不変（進化で姿・大きさが変わるため。着せ替えは主人公＝§9）。

## 7-2. そだった ちから（そだちのもり）… 済（2026-07-16 刷新）

- 5つのちから（順次=じゅんばん／反復=くりかえし／分岐ほか=かんがえる／基本操作=キーボード／創造=つくる）の到達度を、**もりの5本の木**として可視化（bg_mori.webp の上に%座標・足元は底基準アンカーで地面に乗せる）。
- **木の形が概念を表す**: じゅんばん=節が積み上がる／くりかえし=つるが巻く／かんがえる=枝が分かれる／キーボード=実が列に並ぶ／つくる=いろんな花。
- **4段階**（たね／め／わかぎ／おおきな き・しきい値 0/10/40/80=初期値）を**木の大きさ**で表す。**%は出さない**（設計方針: 保護者向けの真面目なバーを子どもに出さない）。称号・絵文字アイコンは廃止。
- **タップで詳細**: 段階の一言（「わかぎに なったよ」＋次への一言）＋**できるように なったこと**（段階ごとに増える箇条書き・たねは「あそぶと ここに ふえるよ」）＋どこで そだつか＋その島への導線ボタン。
- **伸びたら次に訪れたとき祝う**（`save.powers.prev` の差分検出・段階アップは強め・初回は静かに基準化）… 従来の演出ロジックを維持（中の絵が木になっただけ）。
- 到達度の計算（既存 computePowers・保護者画面と同式）は不変。保護者向けの全体像（花びら/レーダー）は別件。

- **コイン獲得＝その日ごとの進歩ベース … 済（実機FB第2便②・06-Cを更新）**: 1回の座りでは同じ実績から二度払われない（＝荒稼ぎ不可）が、日をまたげば同じ実績が再び払うので、毎日遊べば貯まる（反復学習の動機）。記録（生涯ベスト・ずかん）は保持。おえかきのみ1日6枚の上限あり。時間配布・ログインボーナスは無し（遊ばなければ0枚）。上限は設けない（上限は「もう終わり」の合図になり反復を止めるため）。当日基準は `coinDay`（日付が変わると作り直し・既存セーブは初回アクセスで自動生成＝移行不要）。着せ替え/舞台の価格（06-C）は不変。
