## 1. このプロジェクトは何か（ひとことで）

小学生向けの、**育成ゲームで続けたくなるビジュアルプログラミング学習システム**。
画面上でブロックを組んで思考力を育て、相棒モンスターを育てながら、順次・反復・
分岐を体で覚える。ブラウザで動き、タブレットからURLで開くだけで遊べる。

- 位置づけ: **Scratch Jr 相当の自作教材**（低学年）→ 将来 Scratch〜コード入門相当（高学年）へ拡張
- 公開: https://kanda-houtokukai.github.io/programming-land/ （GitHub Pages・無料）
- リポジトリ: ~/programming-land（GitHub: kanda-houtokukai/programming-land）

## 2. めざす姿（ビジョン）

「密度の高い教育システム」。単発ゲームの寄せ集めではなく、**学習の筋道（順次→
反復→分岐→変数→イベント→自由制作→コード）が育成の進行と一体化した、
低学年から高学年まで一本で伸びる学習環境**。有料サービスに並ぶ内容と継続性を、
無料・オフライン可・実在IP非依存で実現する。

## 3. 対象と前提

- 一次対象: 小1〜2（現行）。二次対象: 小3〜6（拡張計画）。
- 利用環境: 自宅のPC・タブレットのブラウザ。オフラインでも崩れない。
- 利用者: 子ども複数（プロファイルで分離・最大4人）＋見守る保護者。
- 制約: サーバー/DB/アカウント/課金/通信対戦は作らない。記録は端末内＋書き出し復元。

## 4. ゆずれない原則（設計判断の憲法）

1. **教育が主・ゲームは動機づけ**。育成や演出は、学びを続けさせるための手段。
2. **実在キャラクター・名前・素材は使わない**。オリジナルのモンスター・絵文字・自作SVGのみ。
3. **ひらがな中心・効果音・ステッカー風UI**（低学年の可読性）。学年が上がったら段階的に漢字を解禁。
4. **データ駆動**。ステージ・クイズ・課題・成長曲線は本体コードに埋め込まず、データ／設定ファイルに分離。
5. **量産物は生成＋全数自動検証**。手作業で大量ステージを作らない。検証FAILは公開しない。
6. **記録は消えない設計**。端末内保存＋バージョン管理＋書き出し/読み込みで移行可能に。
7. **プレビュー成功≠完成**。公開URLをPC・タブレット実機で確認して初めて完了。

## 5. 設計書の地図（このリポジトリの正本群）

| ファイル | 役割 | いつ読む |
|---|---|---|
| **project-charter.md**（本書） | 全体の見取り図・原則 | 最初に必ず |
| **education-curriculum.md** | 何を・どの学年で・どの順に教えるか（学習設計） | 機能を足す/難易度を決める時 |
| **roadmap.md** | フェーズ計画・現在地・完了条件 | 次に何をやるか決める時 |
| **feature-spec.md** | モード別の機能仕様・データ構造・成長曲線 | 実装・仕様変更の時 |
| **progland-handoff.md** | 台帳（現在地サマリ＋経緯アーカイブ・教訓） | セッション開始時に必ず |
| **progland-実装指示書.md** | Codeへの実装指示（P0で作成済み・随時改訂） | Code着手前 |

※ handoff と実装指示書はCode側で既に運用中。本書と curriculum / roadmap / feature-spec を
　今回追加する。内容が重複したら **handoff=いま何をしているか、charter/curriculum/roadmap/spec=
　どう作るべきかの恒久設計**、と役割で棲み分ける。
