## §1. 追加する3枚

| 型 | `label`（棚・6文字以内） | `long` | カテゴリ | ピル |
|---|---|---|---|---|
| `goal` | **ゴール** | ゴールに ついたら | きっかけ（hat） | あいて |

いずれも **gamelab 専用**（`GAMELAB_PALORDER` のみ・studio の `PALORDER` には入れない）。

### 1-1. `goal`「ゴールに ついたら」— ★設計判断（Chat が決めた）

正本 §1 A-3 は「実装が実質同じになるなら `bumpTarget` と同居でよい」と**判断を保留**していた。**カードは別に作る**。

- **挙動の実装は `bumpTarget` と共通でよい**（指定した相手キャラに重なったら発火）。エンジンに新しい概念を増やさない。
- **ただしカードは別に立てる。** 理由:
  - めいろで「**ぶつかったら → クリア**」と書かせるのは、子どもの言葉として意味が通らない。ゴールは「たどりつく」もので「ぶつかる」ものではない
  - きっかけカテゴリの同じ列に並ぶだけなので、増えても棚は散らからない
  - 実装が共通なら保守の負担はほぼ増えない
- **`desc`（ながおしのせつめい）で用途の違いを言葉にする**こと。挙動が同じでも、使いどころが違うと伝われば十分。
  - 例）ゴール＝「ここに たどりついたら クリア！ めいろの ゴールに つかおう」
  - 例）ぶつかったら＝「ふれたら うごきだす。てきや アイテムに つかおう」
- ⚠️ **実機で「同じことをする2枚がある」と混乱するようなら、`goal` を落として `bumpTarget` に一本化する。** その判断は神田さんの実機確認で行う。実装は共通なので、落とすのは容易なはず。

### 1-2. `chase`「おいかける」（うごき・ピル＝あいて）

- 指定した相手キャラに**1拍ごとに1マスずつ寄る**。
- `ずっと → おいかける[主人公]` で鬼ごっこ。**ランダム頼みから脱却する主役カード**。
- 相手と同じマスに着いたらそれ以上寄らない（重なったまま暴れない）こと。

### 1-3. `fall`「ふってくる」（うごき）

- 1拍ごとに1マス下へ。**下端に着いたら上端に戻る**（ループ）。
- `ずっと → ふってくる` でリンゴが降り続ける。`ぶつかったら[主人公] → スコア＋1` で落ちものキャッチが成立する。
- 上端に戻るときは**横位置をランダムにするかどうか**を Code が判断し、**理由とともに報告**すること（同じ列に落ち続けるとゲームにならない可能性がある）。

## §2. ★あわせて直す: 相手ピルが名前でなく内部IDを表示している

**既存バグ。b6c 時点で未修正であることを Chat が確認済み。**

- `StudioBlock` 側は `pillText(b, targetName)` で **`targetName` を受け取る作りになっている**（`targetName ? targetName(b.target) : b.target`）。
- ところが **`WorkshopEditor` から `targetName` を渡している箇所が0**。そのため常にフォールバックし、**ピルに `c2` のような内部IDが出る**。
- 現在これに当たるのは `bumpTarget` の1枚だけだが、**区切り②で `goal` と `chase` が加わり3枚になる**。いま直せば3枚まとめて直る。

**`WorkshopEditor` から `targetName`（cid → キャラの表示名を返す関数）を渡すこと。** 該当キャラが消えている場合の表示も決めて報告すること（「だれか」に落とす等）。

## §3. 区切り①で得た教訓（必ず踏襲）

1. **CSS のテンプレートリテラルより前に定数を宣言する。** `STUDIO_CSS` の中で `${定数}` を参照するのに宣言が後ろにあると、**TDZ でモジュール読込が落ちて画面が真っ黒**になる。`npm run verify` は node で走るため**この種の実行時エラーを拾えない**。**deploy 前に必ずブラウザで表示を確認**すること。
2. **gamelab 専用の CSS は `.gl` スコープに限定する。** `.studio-root.big` などは studio と共有。区切り①で一度これに当たり、`.big.gl` に直している。
3. **新カードの `w` は既存値（206 等）を再利用する。** 区切り①はこれで `geometry.paths` の増加を回避できた。
4. **棚のラベルは6文字以内。** 今回の3枚は3〜5文字なので問題ないが、実装後に実フォントサイズを測って**12px を下回るものが無いことを報告**すること。
