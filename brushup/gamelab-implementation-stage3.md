## §1. 3-A：そうさ（プレイヤー操作）

### A-1. `dpad`「じゅうじキーで うごかす」（そうさ・ティール）
- **形**：body（きっかけではない）。主人公キャラのスクリプトに置く。ピルなし。
- **挙動**：この カードを持つキャラが**遊ぶ画面の十字キーで動かせる**ようになる（＝「操作可能フラグを立てる」カード。1回実行で有効化・以後ずっと有効）。`はたが おされたら → じゅうじキーで うごかす` の形で使う。
- **UI（★A案で確定）**：**常に4方向・ステージの外（下）に固定表示**。画面に重ねない・ゲームによって形を変えない（低学年は「毎回同じ場所に同じキー」が最優先）。`mode.isGame` かつ **操作可能キャラが存在する時だけ**表示。
  - 押しっぱなしで**連続移動**、指を離すと停止（`dpad-play-mock.html` の手触りが基準）。`pointerdown/up/leave/cancel` で保持状態を管理。
  - 全画面（ひろげる）でも同じく下に表示。編集画面の上演エリアでも同様。
- **★拍の扱い（重要）**：本体は1拍=400ms だが、**操作は拍を待たずに即動かす**（待つと明らかにもっさりする）。実装方針：操作による移動は**エンジンの拍とは別に、押下中は短い間隔（例 ~100ms 相当）でグリッド移動**させる。移動は既存のグリッド移動ロジックを再利用し、**盤の範囲でクランプ**。
- **他カードとの関係**：操作移動と、同じキャラのスクリプトによる移動（`みぎへ すすむ` 等）は競合し得るが、**後から来た方を優先**で単純化してよい（低学年が両方置くことは稀）。

### A-2. `tapMove`「タップした ところへ」（そうさ）
- **挙動**：この カードを持つキャラは、**遊ぶ画面をタップするとその方向へ向かって進む**（UIは増えない）。到達したら停止。
- **既存 `tap`（タップされたら）との衝突**：`tap` はキャラ自身がタップされた時、`tapMove` は**背景（ステージ）がタップされた時**。ヒットテストで分離する（キャラの上をタップ＝`tap` 優先）。
- 拍の扱いは A-1 と同じ（待たずに動く）。

### A-3. `goal`「ゴールに ついたら」（きっかけ・hat）
- **挙動**：指定した場所（ゴール）に**着いたら発火**するトリガ。
- **ゴールの決め方**：新概念を増やさないため、**「相手キャラの位置に着いたら」**＝ピルで相手を選ぶ方式（`bumpTarget` と同じUI）にする。旗キャラ等をゴールに置く運用。
  - ※`bumpTarget`（ぶつかったら）との違いは、**自分が動いて到達した時に発火**する点。実装が実質同じになるなら、**設計を単純化して "ゴール＝指定キャラに到達" として bumpTarget と同居**でよい（迷路が作れることが目的）。
- 使い方：`ゴールについたら → スコア＋1` や、クリア条件と組み合わせて迷路のゴール。

### A-4. `chase`「おいかける」（うごき・ピル＝あいて）
- **挙動**：1拍ごとに、**指定した相手キャラの方向へ1マス近づく**（縦横のうち差が大きい方へ寄せる単純追尾で十分）。
- 使い方：`ずっと → おいかける[主人公]` で鬼ごっこ。**ランダム頼みから脱却する主役カード**。

### A-5. `fall`「ふってくる」（うごき）
- **挙動（1枚にまとめる）**：**上端のランダムな位置に出る → 1拍ごとに下へ1マス → 下端に着いたらまた上端のランダム位置へ**（くり返し）。＝落ちものを1枚で表現。
- 使い方：`ずっと → ふってくる` でリンゴが降り続ける。`ぶつかったら[主人公] → スコア＋1` で落ちものキャッチが完成。
- ※「取り逃した」判定が要る場合は段階4のロジックで。段階3では**下端で上に戻るだけ**でよい。

## §2. 3-C：こうぐだな再構成（29種・3-A と同時）

基準＝`palette-29-structure.html`。**gamelab のみ**（`mode.isGame` / `.studio-root.gl`）。studio のパレットは現行のまま。

1. **`bump` を `GAMELAB_PALORDER` から削除**（`bumpTarget` に一本化）。**DEFS からは消さない・studio の `PALORDER` にも残す**（studio は不変）。既存の gamelab 作品に `bump` があっても壊れないよう、**エンジンは `bump` を引き続き解釈する**（パレットに出さないだけ）。
2. **新カテゴリ「そうさ」**を追加：色はティール `#2FB4A6`（edge `#1B8478`）＝既存6色と混ざらない。`COL` に追加。
3. **`GAMELAB_PALORDER` を「ゲームで使う頻度順」に組み替え**（モック準拠）:
   - きっかけ（hat / tap / bumpTarget / goal）
   - **そうさ**（dpad / tapMove / jump※3-B）
   - かず（scoreUp / scoreDown）
   - せいぎょ（forever / repeat / wait）★上位へ
   - うごき（moveRand / bounce / chase / fall / clone※3-B / move / moveL / moveU / moveD / jump移動 / spin / home）
   - みため（hide / show / grow / shrink）
   - おと（sound）
4. **カテゴリ開閉**：見出しタップで開閉（モックの挙動）。初期状態は**みため・おと を閉じ、他は開く**。開閉状態は**セッション中の保持で十分**（保存不要）。
5. カテゴリ見出しに**枚数バッジ**（モック準拠）。

## §3. 3-B：重い2枚（3-A が実機OK後）

### B-1. `jumpable`「ジャンプできるように」（そうさ）
- **1枚にまとめる**：重力＋落下＋着地＋ジャンプを**このカード1枚**で有効化（Scratch なら4〜6枚ぶん）。
- 挙動：このキャラに**重力がかかる**（何もしなければ下へ落ちる）／**地面や足場キャラの上で止まる**／**じゅうじキーの上でジャンプ**。
- 足場：新概念を足さず、**「キャラを並べて足場にする」**で代用（design 判断）。地面＝盤の最下段。
- 実装が重いので、**単体で先に実機確認**してから他と組み合わせる。

### B-2. `clone`「ぶんしんを だす」（うごき）
- **挙動**：自分と同じキャラを**その場に1体増やす**（弾・落ちもの・湧く敵）。
- **上限を必ず設ける**（例：同時20体・超えたら古い順に消す）＝暴走防止。分身は**元のスクリプトを引き継ぐ**。
- 使い方：`ずっと →（まつ→ぶんしんを だす）` で弾幕、`ぶんしん＋ふってくる` で大量の落ちもの。

## §5. スキーマ / 検証 / ゲート

- **`SCHEMA_VERSION`**：新カードは作品内のブロック型なので**上げない**。ただし**操作可能フラグ等をキャラ状態に持つ場合も保存不要**（スクリプトから毎回導出する）。`gameConfig` も変更なし。

- **検証**：エンジン単体に `chase`（寄る方向）・`fall`（下端で上に戻る）・`clone`（上限）・`jumpable`（落下と着地）のケース追加。`verify-gamelab.mjs` に新みほんの妥当性（新カード・ゴール/相手の実在）。

  1. じゅうじキーで**もっさりせずに**動く（拍を待たない）・押しっぱなしで連続移動・全画面でも出る。
  2. おちものキャッチ／めいろ／おにごっこが**作れて遊べる**。

  4. **studio が完全無変化**（`bump` も含めて従来どおり）。
  5. `npm run verify` 全PASS・回帰GREEN・コンソールエラーゼロ。
