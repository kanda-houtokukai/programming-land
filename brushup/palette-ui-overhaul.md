## §1. レイアウトの規則（gamelab・studio 共通）

**原則：画面に最大限収める**（既存の `min()`＋`100dvh` の思想と同じ。縦スクロールは こうぐだな の中だけ許す）。

1. **こうぐだなは つねに横2列**。棚の幅が変わってもカードの幅を自動計算して2列を保つ。
   - `カード幅 = floor((こうぐだなの実はば − すきま7px − 1) / 2)`
   - **実はば＝スクロールバーを除いた幅**（`palscroll.clientWidth`）から取る。**`scrollbar-gutter: stable` ＋ 細いスクロールバー（5px）**を指定して幅を安定させる。※これを怠るとPCで1列に折り返す（モックで実際に起きた）。
   - カードは `flex:0 0 auto`（伸び縮みさせない）。
2. **作業エリアとプレビューは、のこりを半分ずつ**（`flex:1 1 0` を両方に）。
3. 幅は**％で持つ**（px固定にしない）＝PC・タブレットどちらでも同じ配分になる。
   - 初期値：**こうぐだな 24%／作業 38%／プレビュー 38%**（実機モックで決定。実機で微調整可）

## §2. 大きさ（実機モックで決めた値）

| 項目 | 値 | 備考 |
|---|---|---|
| こうぐだなのブロック | **76%**（高さ約36px） | 2列に収めるため小さめ |
| 作業エリアのブロック | **98%**（高さ約47px） | 数字をタップして変えるので大きめ |
| こうぐだなの文字 | **カードの中で自動最大化・上限16px** | カードごとに最大サイズを計算（短い名前ほど大きくなる） |
| 作業エリアの文字 | 14px | |

**文字の自動最大化**：`文字サイズ = min(16, (カード幅 − ラベル開始X − 8) / 文字1つあたりの幅)`。canvas の `measureText` で実測して求める（モック `fitFont` 実装が基準）。

## §3. カードの見え方（★ここが新しい）

- **こうぐだな**：**みじかい名前のみ・数字（ピル）は出さない**。全カードが同じ幅で揃う＝探しやすく、文字も大きくできる。
- **作業エリア**：**ながい名前ではなく みじかい名前のまま**、ただし**数字（ピル）が出る**＋ブロックが大きい。
  - ＝「棚では選ぶだけ／作業エリアで調整する」という役割の違いが見た目で伝わる。
- **ドラッグ中**：ゴーストは棚のサイズで始まり、**作業エリアに入った瞬間に大きくなって数字が出る**（モック実装が基準）。この変化が「ここで使える形になる」合図になる。
- **ながい名前と説明は、ながおしの ふきだしでのみ表示**（§5）。

## §4. こうぐだなの構成

- カテゴリ見出しは**タップで開閉**。初期は **みため・おと を閉じ、他は開く**。
- 並び順は**ゲームで使う頻度順**（gamelab）：きっかけ → **そうさ** → かず → せいぎょ → うごき → みため → おと。
- **そうさ**カテゴリ新設（色＝ティール `#2FB4A6` / edge `#1B8478`）。
- studio は現行のカテゴリ・並びのまま（§7）。

## §5. ながおし操作（tap-to-copy を廃止）★中核

**現状の廃止**：パレットのカードを `pointerdown` で即ドラッグ開始＆コピーする挙動をやめる。**棚の上ではタップでもながおしでもコピーされない**。

**状態遷移**（モック実装が正・しきい値は実機で微調整可）:

| 指の動き | 起きること |
|---|---|
| **軽く触って離す** | **何も起きない**。初回だけトースト「ながおしすると せつめいが でるよ」 |
| **150ms 押し続ける** | カードが少し大きくなり（`scale 1.14`）、**ふきだしで「ながい名前＋せつめい」**が出る |
| ながおし成立**前**に指が動く（7px超） | ながおし中止 → **縦スクロールに譲る**（`touch-action: pan-y`） |
| ながおし成立**後**に動く（6px超） | **ドラッグ開始**。ふきだしを消してゴーストが指に追従。作業エリアが光る |
| 作業エリアで離す | **そこに置く**（トースト「おいたよ！」） |
| 作業エリア外で離す | **取り消し**（トースト「とりけし」） |
| **指を離す（いつでも）** | **ふきだしは消える**＝「押している間だけ説明が出る」 |
| `pointercancel`（ブラウザがスクロールを取った） | すべて中止・ふきだしとゴーストを消す |

**確定値**：ながおし **150ms**／ながおし前の許容 **7px**／ドラッグ開始 **6px**。
※150msは軽いタップ（80〜120ms）に近く、触っただけで出ることがあるが、**離せば消えるので害はない**（反応の軽さを優先した判断）。

**実装上の注意**
- ながおしのタイマーは `pointerdown` で開始し、`pointermove`（しきい値超）・`pointerup`・`pointercancel` で必ず解除する。
- ドラッグ中は `setPointerCapture` を使う。
- 既存の「トリガは2枚目を拒否（プルッ）」等の判定は、**ドラッグ確定時**に行う（b5v で入れた作法と同じ）。

## §7. つくるスタジオへの適用

- **ながおし操作（§5）は studio にも入れる**（誤コピー防止と、説明が読めることは低学年ほど必要）。**せつめい文は上表を共用**（studio にある18種ぶん）。
- **レイアウト（§1〜§4）を studio にも適用するかは神田さんの最終判断**。適用する場合は studio も2列・短い名前・数字は作業エリアのみになる（＝現在の「枠＋カテゴリ名つき2列グリッド」は廃止）。
  - 適用しない場合：studio は現行の見た目のまま、**§5の操作だけ**入れる。
  - **Code は、この判断を神田さんに確認してから studio 側の見た目に手を付けること。**

## §8. ベースライン・検証（重要）

- 本作業は **`G` の使い方（棚と作業エリアで倍率が違う）・ラベル文字列・チップ配置**に触れるため、**凍結ジオメトリのベースラインは変わる**。`palette-fixes.md` と同じ手順で:
  1. 変更適用 → 回帰の FAIL 内容を確認
  2. `--update` で再取得
  3. **diff が「サイズ・ラベル関連」だけであることを機械確認**。**エンジンのトレース（732イベント）は1バイトも変わってはいけない**（＝挙動不変）。変わっていたら**止めて原因究明**。
- `npm run verify` 全PASS・ビルドOK・コンソールエラーゼロ。
- **実機ゲート（iPad横向き＋PC）**:
  1. こうぐだなが**必ず2列**（PCでも折り返さない）／棚の幅を変えても2列を保つ
  2. **軽く触ってもコピーされない**／150msで説明が出る／**離すと消える**／引っぱると作業エリアで大きくなり数字が出る／作業エリア外で離すと取り消し
  3. **縦スワイプで棚がスクロールできる**（ながおしと競合しない）
  4. 作業エリアとプレビューが半分ずつ・画面からはみ出さない
  5. studio でも §5 の操作が同じように効く（見た目は §7 の判断どおり）
