# 実装指示書: ショップをRPGの店にする

作成: 2026-07-06 ／ 設計の正本: 【決定メモ05】
モデル: Opus 4.8（大量生成なし）

現状のショップ（白いカードに商品が並んだリスト）を、RPGの道具屋に刷新する。
店内に入ると店員（ひげのおじさん）がカウンターの向こうにいて、話しかけると
「どうぐ」か「きせかえ」を選び、商品リストへ。おうち（メモ04）と同じ手法。

---

## 使う素材（用意済み・src/assets/へ）
- shop-interior.webp（店内背景・1600×1073・141KB）。カウンター＋ひげのおじさん店員＋
  後ろの棚に商品が描き込まれている。店員は中央。

## 1. 構成（2段階）

### 段階1: 店内の風景（店員に話しかける）
- 店内背景（shop-interior.webp）を表示。
- **店員（中央のおじさん）をタップ**すると話しかける。タップ領域は中央（約 50%, 42%）に
  大きめに設定（子どもが押しやすいよう店員の体〜手のあたりを含める）。
  ブラウザで店員の絵に合わせて微調整。
- 店員に、タップできると分かる控えめな演出（うっすら光る/軽い動き＋タップで反応）。
  ※【メモ04】のタップ表現方針と同じ「控えめ・世界観優先」。
- 話しかけると、店員の近くに吹き出し＋選択肢:
  - 吹き出し（店員の第一声）: 「いらっしゃい！ なにを かいにきた？」（※文言は下記、要確定）
  - 選択肢: 「🧃 バトルの どうぐ」／「🎩 きせかえ」／「やめる」
- 「やめる」でワールドマップ（または元の画面）に戻る。

### 段階2: 商品リスト（今の画面を流用）
- 選んだカテゴリの商品リストを表示（今のカード画面をそのまま流用）:
  - バトルのどうぐ: かいふくドリンク10／ヒントメガネ15／パワーバンド15／まもりのたて15
    （各所持上限3）
  - きせかえ: 主人公（探検家）の着せ替えアイテム8種（6スロット）＋バトルのぶたい2種。
    詳細・価格は `replacement_instructions_for_code.md` §B・`dressup_system_design.md`
    §4データ構造を参照。バトルのぶたいは「ジャングルのぶたい」「だいちのぶたい」
    （各50枚・`battle_stage_jungle_nature_prompts.md`参照）。
- 買うと店員が反応（「まいど！ ありがとう」等の吹き出し・軽い演出）。
- 商品リストから「もどる」で段階1（店内）に戻る。カテゴリを選び直せる。
- バトルのどうぐの価格・所持上限・効果・コイン消費は現状のまま変えない。

## 2. 店員の吹き出し文言（RPGの店員らしく・要ユーザー確定）
案（親しみやすい低学年向け）:
- 第一声: 「いらっしゃい！ なにを かいにきた？」
- どうぐ選択後（任意）: 「バトルで つかう どうぐだね！」
- きせかえ選択後（任意）: 「おしゃれを えらんでね！」
- 購入時: 「まいど！ ありがとう」
- コイン不足時: 「コインが たりないみたいだ。また おいで！」
- やめる時: 「またね！」
※文言はユーザーと最終確認して調整。

## 3. ヘッダー・戻る（メモ01/03と整合）
- ショップは、ワールドマップの「おみせ」から入る。段階1（店内）が入口。
- 「もどる」は1階層ずつ（メモ03の原則）: 商品リスト→店内→ワールドマップ。
- ヘッダーの相棒・⭐・🪙は維持（コイン残高が見える）。

## 4. やらないこと / 注意
- 店員・カウンター・棚は背景に描き込み済み。動かさない・別画像で乗せない。
- 商品リスト（段階2）のうち「バトルのどうぐ」（4種の効果・価格・所持上限）は
  変えない。「きせかえ」は上記修正のとおり主人公用に総入れ替えする
  （これは今回変更する対象。店の入口を足すだけ、ではない点に注意）。
- 「あそびかた」「おうちの方へ」は現状どおり付ける（ショップ用の文言は既存を流用）。
- verify全PASS。バージョンを上げてデプロイ、ユーザーが実機確認。

## 5. 完了条件
- ショップが店内風景になり、店員をタップ→「どうぐ/きせかえ」選択→商品リスト、の流れ。
- 買うと店員が反応。もどるで店内→ワールドマップと1階層ずつ戻る。
- 商品の価格・効果は不変。既存機能不変・verify全PASS。実機確認。

## 6. 関連
- おうち（メモ04）と同じ「背景＋タップ→次へ」の手法。まとめて実装すると効率的。
- タップ表現は控えめ（メモ04と同方針）。
