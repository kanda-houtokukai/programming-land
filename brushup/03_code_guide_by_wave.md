# Code実装ガイド — 波ごとの「使うファイル」と「渡す指示」

作成: 2026-07-06
目的: 全ファイルをローカルに置いた状態でも、Code が混乱しないよう、
      **どの波で・どのファイルを使い・どう指示するか**を明示する。

## 使い方
- 今日のパックの42ファイルは、一度に全部ローカル（~/programming-land/ 等）に置いてよい。
- ただし **Code への実装指示は、下の波の順で、1波ずつ**出す。
- 各波の「Codeに渡す指示」をそのままCodeに貼り、「使うファイル」に挙げたものだけを
  その波で src/assets/ 等へ移す。他の波のファイルはまだ触らないようCodeに伝える。
- 各波は完了・実機確認してから次の波へ。大きい波（第2波・第4波）は波の中でも段階を切る。

---

## 第0波: 開発途中の実機確認（新規実装なし）
**使うファイル**: なし（既に公開済みのものを実機で触るだけ）
**やること**: そだったちからフェーズ1・ワールドマップ現状を、実際に子どもと触って問題洗い出し。
**Codeに渡す指示**:
> 新規実装の前に、現状を実機確認します。そだったちから（ホームの5つの力）と
> ワールドマップの各エリアを、実際に触って不具合や気になる点を洗い出します。
> ここで新たな修正が出たら、その対応を優先します。（Code作業は基本なし・確認結果待ち）

## Chat（次のClaude）から受け取ってから進める箇所（重要）
以下は「Chatが用意 → それを受け取ってCodeが実装」の順。該当波の前にChatに依頼:
- 第2波②の前: 起動時プロファイル選択画面の背景（港）＋港/ふねアイコン。

## 全波共通
- 既存機能を壊さない・verify全PASS・書き出し読み込み後方互換を各波で担保。
- 拡張の器として作る（データは設定ファイル集約・場所とゲーム分離・拡張枠を残す・共通部品化）。
- 他の波のファイルはその波では触らない（混乱防止）。
