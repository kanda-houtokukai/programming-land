## 0. 確定事項（2026-07-19・神田さん承認）

1. **新モード**として実装する（スタジオ拡張ではない）。カードエンジン・エディタ・実行モデルは
   共通部品化して共有するが、スタジオの憲章「勝ち負けも正解もない自由制作」は不変のまま守る。
2. 名前は「**ゲームこうぼう**」。マップ短ラベル「**ゲーム**」（メモ02の3階層命名に準拠）。
3. 場所は**右中(69.5, 34.13)**。現1枚目のじゅんびちゅう看板を建物に置換（studio-design §3 の
   「第2期用の予告」どおりの開店）。中央上(47.3, 29.25)の看板は帯C最終枠として温存。
4. v1スコープ: 「もし〜なら」容器ブロックは**見送り（器のみ・第2期）**。分岐は
   「ゲームのおわり条件」と「ぶつかったら」イベントが担う。変数（スコア）を確実に据える。

### 棲み分け（スタジオとの関係・設計の背骨）
| | つくるスタジオ | ゲームこうぼう |
|---|---|---|
| 何を作る | 動く作品（見せるもの） | 遊べるゲーム（遊ばせるもの） |
| 勝ち負け | ない（正解のない自由制作＝憲章） | ある（スコア・クリア・ゲームオーバー） |
| 学びの主役 | イベント・並行・くりかえし | 変数（スコア）・ルール設計・テストと調整 |
| 帯 | 帯A→Bの入口 | 帯Bの核（→帯Cへの足場） |
| 棚 | フィルムだな | カセットだな（相互変換なし） |
| 子どもの動線 | まず作り方を覚える | 覚えた部品でゲームにする |

重なりはエンジン（裏側）だけ。子ども・保護者から見える体験は別物として設計する。

## 1. このモードは何か（位置づけ）

スタジオが「**動く作品**を作る道具」なら、こうぼうは「**遊べるゲーム**を作る道具」。
差分はただ1つ——**おわり（クリア／ゲームオーバー）がある**こと。スコアが増え、目標に届けば
おいわい、時間切れや接触で終わる。作った本人も家族も**プレイヤーとして遊べる**。
これが「スタジオを超える」の中身: 作品が"見せるもの"から"遊ばせるもの"へ。

Scratchの丸ごと移植はしない（スタジオと同判断）。**スタジオで習得済みの部品**（カード操作・
きっかけ・くりかえし・並行）の上に、**スコア（変数）とゲームの器**だけを足す制約のある設計。

## 2. 育てる力（カリキュラム接続）

1. **変数**（柱⑤・本作初実装）。スコア＝「数をおぼえて増やす・目標と比べる」の最初の実体。
   帯B（小3〜4）の核であり、本モードの教育的な主役。
2. **ルール設計**＝意図の実現の発展形。「どうなったら勝ち? 何をしたら点?」を子どもが決める
   ＝要件定義の芽。
3. **テストと調整**。自分で遊ぶ→難しすぎ/簡単すぎ→数値を直す＝パラメータ調整とデバッグの往復。
4. **イベント・並行・反復の応用**（スタジオ習得分を"ゲームの文脈"で使い直す）。
5. **保護者への翻訳**: 「点数の仕組み（変数）と勝ち負けのルールを自分で組んだ」
   ＝Scratchで行う学習そのもの、と保護者ガイドで明言できる。

そだったちから: 「つくる」の第2水源＋新規要素として変数使用を係数化（powers.jsデータ側）。

## 3. 場所・名前・マップ

- 右中(69.5, 34.13)の看板を撤去し「**こうぼうの建物**」を設置（tall推奨・要生成: 歯車＋
  ゲームパッド意匠・スタジオ建物と同画風）。加工規約は studio-design §10-2 を全面踏襲
  （縦余白ゼロ・左上光源・**マップ右側=左右反転**・透過256px PNG）。
- 中央上の看板（flip版）は残す＝「まだ次がある」予告として機能し続ける。
- 幅ゲート: スタジオと同じ「タブレットか パソコンで あそんでね」（CSS幅のみ・JS向き検出禁止）。

## 4. ゲームの器（スタジオとの最大差分・ブロックの外側）

作品＝「ステージ＋キャラ＋プログラム」（スタジオと同形式）＋ **gameConfig（ゲームせってい）**。
せっていはブロックでなく**専用パネルで選ぶ**（低学年に文法を増やさない）:

- **スコアひょうじ**: オン/オフ。オンなら上演画面の上部に「⭐ 0」風の大きなスコア。
- **クリアじょうけん**（1つ選択）:
  | 種類 | パラメータ | 挙動 |
  |---|---|---|
  | スコアが たまったら | 5〜50（5刻み） | 到達した瞬間おいわい画面 |
  | じかんが おわるまで | 10〜60秒（10刻み） | 残り時間を表示。0でけっか画面（スコア発表） |
  | クリアなし（じゆう） | — | スタジオ同様に終わらない（■で終了）。移行用の逃げ道 |
- **ゲームオーバー**（オプション・オン/オフ）: 「**ばくだんタッチ**」＝指定キャラを
  「さわったら まけ」に指定（キャラ一覧でドクロ風マークを付ける）。よけゲームの核。
- **おいわい/ざんねん画面**: 紙吹雪＋スコア＋「もういちど」「なおす」。ゲームオーバーは
  責めない文言（「おしい! もういちど?」）＝「失敗を罰しない」原則の維持。

## 5. カード（スタジオ18種＋こうぼう専用の追加分）

スタジオ全18種はそのまま使える（同一エンジン）。追加は最小限:

### かず（新カテゴリ・きいろ系・変数）
| カード | パラメータ | 挙動 |
|---|---|---|
| スコア +（ふやす） | 1〜5 | スコアを足す（1拍）。+の瞬間スコア表示がぽよん |
| スコア −（へらす） | 1〜5 | スコアを引く（下限0・1拍） |

### うごき（青・ゲーム用の追加）
| カード | パラメータ | 挙動 |
|---|---|---|
| ランダムに うごく | なし | 上下左右のどれかへ1マス（1拍）。「ずっと」と組むと徘徊 |
| はねかえる | なし | 進行方向を反転して1マス（端で自動反転・1拍）※studio-design §12の候補を実装 |
| （相手指定）〜に ぶつかったら | 相手キャラ選択 | きっかけの拡張。既存「ぶつかったら」に相手ピルを追加（だれでも/特定キャラ） |

- 追加は**5枚以内**に抑える（1画面のたなに収まる・認知負荷）。全カードは
  `data/studio-blocks.js` へのデータ追加＝スタジオ設計の「新カード=データ追加」の器をそのまま使う。
- **もし〜なら容器・メッセージ・座標・キー操作は v1 やらない**（§0-4・§13）。

### 遊びが成立する最小レシピ（設計検証・この3つが作れれば器は正しい）
- **あつめゲーム**: リンゴ役キャラ「タップされたら→スコア+1→きえる→まつ→でる」×3体、
  クリア=スコア10。
- **よけゲーム**: 敵「ずっと→ランダムにうごく」、ばくだんタッチ=敵、クリア=じかん30秒。
- **キャッチ**: 主人公はタップ移動（既存タップきっかけ+うごきで代替）、相手指定ぶつかりで+1。

## 6. 実行モデル

スタジオと同一（TICK 400ms・並行・発光・■=カット）。追加はゲーム層のみ:
- スコア/タイマーは**ゲームの器が管理**（キャラの外・描画は上演画面のHUD）。
- クリア/ゲームオーバー判定は毎拍の最後に評価（同拍に+3でまたぎ到達しても発火）。
- 発火したら全キャラ停止→おいわい/ざんねん画面。「もういちど」=初期化して再スタート。
- 編集画面の▶はテストプレイ（同じ判定が動く）。「ひろげる」で全画面プレイ＝家族に遊ばせる顔。

## 7. 作品まわり

- 保存形式: スタジオ形式＋`gameConfig`。純データ・サムネはデータからミニ描画（流用）。
- 保存先: `gamelab: { works: [...], draft: null }` をプロファイル単位で追加。
  **SCHEMA_VERSION+1**・デフォルトマージ・roundtrip試験に追加（スタジオと同型）。
- かきかけ自動保存・上限30・名前ひらがな8文字・「カセットだな」（ギャラリー・ゲームカセット風
  フレームでフィルムだなと差別化）・「みほんのたな」＝§5の最小レシピ3本＋1（削除不可・
  リミックスで原本不変・`remixOf`記録・verifyにスキーマ妥当性を載せる）。
- スタジオ作品との相互変換は**やらない**（v1。器の違いで混乱するため）。

## 8. 実装戦略（スタジオを壊さないことが最優先）

- **段階A（共通化リファクタ・機能追加ゼロ）**: StudioEditor/ブロック描画/実行エンジン/棚UIを
  「モード非依存の共通部品＋スタジオ薄皮」に分離。**見た目・挙動は1pxも変えない**。
  verify にスタジオ回帰（みほん4本のシリアライズ一致等）を足して機械保証。【実機確認】
- 以後、こうぼうは共通部品＋「こうぼう薄皮（新カード・ゲームの器・HUD・カセットだな）」として実装。
- スタジオの b5m 下書き修正・幅ゲート等の既知の教訓はそのまま共通部品に引き継がれる。

## 9. 教育接続（獲得は進歩ベース原則の維持）

- XP: 新規作品の初回保存のみ（空作品ガード=クリアじょうけん設定済み＋動くスタック1本以上）。
- コイン: 初回マイルストーンのみ（初ゲーム/はじめてスコアカード/はじめて自作ゲームをクリア/
  みほん初リミックス）。単価は既存レバー（06-C）に揃える。
- バッジ候補3種: はじめてのゲームクリエイター／テストプレイめいじん（自作を10回あそぶ）／
  スコアのたつじん（スコアカード両方を使った作品保存）。
- 着せ替え候補1点: 「ゲームディレクターぼうし」等（全追加カード使用で解放・憲章§4-9）。
- そだったちから: 「つくる」へ加点＋変数使用を新係数。きろく日別log `gamelab`。
- 保護者ガイド骨子: ①スコア=変数（柱⑤）の初学習 ②ルールを決める=要件定義の芽
  ③自分で遊んで直す=テストと調整 ④Scratchとの対応表（この遊び=Scratchのこの学び）。全文Chat執筆。

## 12. やらないこと

- Scratch丸ごと（座標・キーボード操作・自由ピクセル移動・9カテゴリ）。
- もし〜なら容器（v1）・スタジオ作品との相互変換・オンライン共有・作品毎コイン。
- スタジオ本体の挙動変更（共通化は等価変換のみ）。スマホ対応（案内表示に留める）。
- 向き検出JS。確認ダイアログの新設（とりけし常設で代替・スタジオ作法）。
