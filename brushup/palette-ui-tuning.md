作成: 2026-07-22 ／ 状態: 確定（神田さん実機確認・モックで数値決定済み）／ 対象: Code
置き場所案: `brushup/palette-ui-tuning.md`
**基準モック（必ず開く）**: `brushup/palette-mock3.html`（3つのつまみで確定した値が初期値に入っている）
前提: `palette-ui-overhaul.md` 実装済み（**v2.3-b5x**・実機確認済み）。本書はその**微調整のみ**。UIの構造・操作（ながおし150ms→せつめい→ドラッグ）は変えない。

> **実機フィードバック（神田さん・b5x）**
> ① こうぐだなのカードが指定幅いっぱいで**ごちゃごちゃ感**がある → 少し縮めて余白を持たせたい
> ② 作業エリアのカードが**大きく、中の空白が気になる** → 全体的に小さくてよい

## §1. こうぐだな：カードに余白を持たせる

現在は「ぴったり2列」の幅をカードが**フルに使っている**ため詰まって見える。カードを少し縮め、余ったぶんを**両端と列の間に均等配分**する。

```
full  = floor((こうぐだなの実はば − 7 − 1) / 2)     // 今までの「ぴったり2列」幅
カード幅 = floor(full × 0.92)                        // ★92%（実機モックで決定）
slack = 実はば − カード幅 × 2
unit  = max(4, floor(slack / 3))
→ カード置き場の padding-left / padding-right = unit、カード間の gap = unit
```
- **2列は維持**（折り返さない）。`scrollbar-gutter: stable` と `clientWidth` からの算出は b5x のまま。
- 文字の自動最大化（上限16px）も b5x のまま。カード幅が縮むぶん、文字は自動的に少し小さくなる。

## §2. 作業エリア：小さくする＋空白をなくす

### 2-1. 縮尺
- 作業エリアのブロックの縮尺を **86%** にする（b5x は等倍）。
- ※b5x で Code が等倍を選んだ理由（磁石78px 等の凍結値の保護）は妥当だった。今回は**縮尺を変えるので、磁石・接続・ドラッグの当たり判定が縮尺に追従するか必ず確認**すること。追従しない実装なら、**縮尺に比例させる**か、当たり判定だけ従来値のままにする（実機で接続しづらくなっていないかを確認）。

### 2-2. 空白の除去（★根本対応）
b5x で名前を短くした結果、**`DEFS.w` が昔の長い名前のままのカードに、右側の空白**が出ている（実測: `hat` 218px「はた ▶」／`move` 206px「みぎへ」／`sound` 198px「おと」／`moveL` 206px「ひだりへ」／`home` 190px「もとのばしょ」。比較: `hide` 138px「きえる」）。

**方針A（推奨）: 作業エリアのカード幅も「内容ぴったりの自動計算」にする。**
- こうぐだなと同じ考え方で、`ラベル開始X + 文字幅 + ピル幅 + 余白` から都度算出する（モック `canvasW` が実装例）。
- 利点: **今後カード名を変えても空白が出ない**。`DEFS.w` の保守が不要になる。
- リスク: `w` はジオメトリの基準値なので、**磁石・接続・スタック計算が `w` 依存なら影響が出る**。着手前に依存箇所を調査すること（鉄則1）。

**方針B（Aが危険と判断した場合のフォールバック）: 上記5枚の `w` だけを内容に合った値に縮める。**
- 影響範囲は小さいが、今後も名前を変えるたびに手当てが要る。
- **A/B どちらを採るかは Code が調査のうえ判断してよい**が、**判断と理由を報告すること**。

## §3. 折りたたみ見出し：押しやすく（見た目は変えない）

- 見出し行（`.sec-h`）に **`min-height: 34px`** を与え、**押せる範囲だけを広げる**。
- `border-radius: 8px` ＋ **押下時に薄い背景色**（例 `#e4eaf2`）を付けて、反応が分かるようにする。
- **文字サイズ・色・区切り線・ドットは変更しない**（極端に変えない＝神田さんの指示）。

## §4. 変えないもの

ながおし **150ms**／許容7px・ドラッグ6px／こうぐだな **24%**・作業とプレビュー**半々**／こうぐだなの縮尺 **76%**／短い名前のみ・ピルは作業エリアだけ／カテゴリ開閉と初期開閉状態／studio と gamelab 両方に適用（b5x の §7 決定どおり）。

## §5. 検証

- **ベースライン再取得が必要**（§2 で縮尺・幅が変わるため）。`palette-ui-overhaul.md` §8 と同じ手順:
  1. 適用 → 回帰の FAIL 内容を確認
  2. `--update` で再取得
  3. **diff が「幅・縮尺関連」だけであることを機械確認**。**エンジンのトレース（732イベント）は1バイトも変わってはいけない**。変わっていたら止めて原因究明。
- `npm run verify` 全PASS・ビルドOK・コンソールエラーゼロ。
- **実機ゲート（iPad横＋PC・studio/gamelab 両方）**:
  1. こうぐだなが**2列のまま**、カードに余白ができて詰まって見えない
  2. 作業エリアのカードが小さくなり、**「はた ▶」「みぎへ」「おと」に右の空白がない**
  3. **ブロックの接続（磁石）が従来どおり効く**（縮尺変更の影響がない）
  4. 見出しが**押しやすく**なった／見た目は b5x と変わらない
  5. ながおし・ドラッグ・スクロールは b5x のまま
