## 4. この設計で確立し、他の作業にも効く規約（横展開すべき）

このチャットの検証で、**既存アセット全体に関わる規約**が判明した。今後の全アセットに適用する。

1. **縦の余白ゼロ**: 絵は256px canvasの上下端に接する（既存 building_quiz/home/typing は
   すべて上0/下0）。※初回加工で4%余白を付け、マップ上で約7%小さく表示される不具合を
   出したため是正。**この規約はこれまで明文化されていなかった**。
2. **光源は左上**: 既存建物はすべて左光源（quiz +5.9 / home +23.8 / typing +40.1）。
   接地影 `drop-shadow(1px 2px 2px)` が右下に落ちる設定と整合。マップ右側に置く物は
   左右反転すると構図・光源の両方が整う。
3. **マップ座標の換算規約**: `dx = r·sin(θ), dy = −r·cos(θ)`（θ=12時から時計回り）。
   既存の実績値（myhome 0.5pt2時 → 58.43/65.75、puzzle 1.0pt2時 → 85.87/71.5）で
   検算し完全一致を確認済み。今後の位置調整もこの式で計算できる。
4. **フルカラー3Dアイコンは彩度の高い面に直置きしない**: 実測で18個中13個が埋没
   （最悪コントラスト比1.10）。**白チップ台座**（既存の白ピルと同じパターン）で解決する。

→ 1・2・4は `progland-handoff.md` か art guide 側へ、3は WorldMap.jsx のコメントへ
反映する価値がある（反映は Code）。

## 5. 本線側で判断が要ること

1. **合流の順序**: 本線の残件と、つくるスタジオ段階1（storage.js の版上げ）の
   どちらを先に push するか。Code は1レーンなので、実際には順番に流れる。
2. **保留中の品質判断2点**（実装をブロックしない・後から差し替え可能）:
   - 矢印4種（みぎ/ひだり/うえ/した）が色もデザインもバラバラで「家族」に見えない
     → 同一デザインの4方向で再生成するのが望ましい
   - 「でる」が魔法の帽子で、対になる「きえる」（おばけ）と噛み合わず、18個中これだけ黒い
     → おばけが光って現れる絵に再生成すると対になる
3. **第2期の予告看板**: 右中の空き地には「じゅんびちゅう」看板を置く。第2期（イベント型の
   発明ラボ／「もし〜なら」・変数・メッセージ）は器だけ用意済みで、着手時期は未定。
