## 0. 全体の流れ（何が起きたか）

1. 神田さんから「着せ替えシステム設計案」が提示される（相棒でなく主人公を
   着せ替え対象にする／部位固定／レイヤー合成／学習内容と紐づけたアイテム）。
2. Chatが基盤設計（スロット・zオーダー・データ構造）を整理。
3. ベース人物（男の子・女の子、透過PNG）の精度検証 → 脇の下・目・靴・
   首と髪の隙間など、透過処理の技術的な問題を複数発見・修正。
4. 「カバン・ベルトは重ねるだけでは装着に見えない」という神田さんの指摘 →
   正面から見て体に隠れる部分は描かない、という原則を確立。
5. 着せ替えの方向性を神田さんの一声で**0から再構成**（相棒用に考えられて
   いた「シルクハット」「きらきら」等は世界観に合わないため削除。
   「探検家」の世界観に合わせて再設計）。
6. 主人公の服を「探検家風」に刷新（ベスト・カーキ色・ブーツ）。
   → ベース人物を作り直し（6体: 男女×背負わない/キーボード/魔法の本）。
7. 6スロット（head/face/neck/chest/waist/back）を確定。back のみ
   「レイヤー合成」でなく「ベース人物ごと切り替え」方式に変更
   （リュックは装着後の見た目差が大きいため）。
8. face（たんけんメガネ）・neck（たんけんバンダナ）を新設・実測確認。
   バンダナは「体に隠れる部分（首の輪っか）を描かない」原則を2回に
   分けて適用し、機械的トリムでなく**生成し直す**方が自然に仕上がる
   という教訓を得た。
9. Codeがショップ実装に入るタイミングで、バトル消耗アイテム4種
   （絵文字プレースホルダー）もイラスト化。
10. 最終確認で、リュックの肩ひも素材（過去のレイヤー案の名残）が
    もう不要と判明 → 本パッケージから除外し、必要なものだけに厳選。

### むしめがねバッジ（chest・2回目の生成で確定）
1回目はバッジ形状でなく「持ち手つきの虫眼鏡そのもの」だったため再生成。

## 3. スロットのルール（設計の要点）

- 同じスロットは1個まで装備。**違うスロットは自由に組み合わせ可能**
  （リボンをつけたらリボンしかつけられない、という問題を解消する設計）。
- z順（奥→手前）: back(6体切り替えのため対象外) → waist → chest → neck → face → head。
- **体に隠れる部分は描かない**（今回何度も学んだ原則）。カバン・ベルト・
  バンダナ等、体を巻く/背負うアイテムは、正面から見て隠れる部分を
  そもそも絵に含めない。編集でトリムするより、生成し直す方が縁が自然。
- データ構造（`acquire.type`: shop/achievement/default）は
  `docs/dressup_system_design.md` §4 を参照。
