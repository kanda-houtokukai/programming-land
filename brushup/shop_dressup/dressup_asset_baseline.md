# 着せ替え・アイテム作成の基準（レシピ＆確定アンカー）

作成: 2026-07-08 ／ 種別: 恒久リファレンス（今後のアイテム展開の基準）
親: project-charter.md ／ 実装値の正: `src/data/dressup.js`（本書と食い違ったら dressup.js が正）

このファイルは「**この位置・この画風を基本に、今後アイテムを増やす**」ための基準集。
新しいアイテムを作るときは、①共通スタイルでプロンプトを書き、②生成物を後処理し、
③下表のアンカー方式で載せ、④実機で微調整する。個別の生成プロンプト本体は末尾の
「プロンプトファイル」を参照（本書と一緒にリポジトリへ置く）。

---

## 0. 世界観（画風の芯）

- 主人公＝**島を冒険する子ども探検家**。カーキ／アースカラー中心（ベスト・ポケット・ブーツ）。
- Pixar調3Dレンダー・柔らかい光源（正面やや上）・彩度高め・子ども向け教育ゲーム。
- 実在キャラクター・名前・ロゴ・文字は不使用（憲章§4）。

## 1. 共通スタイル（生成プロンプトの共通部分）

新規アイテムのプロンプトは、必ずこの共通部を土台にする（英語でChatGPTへ）:

```
Pixar-style 3D render, soft lighting, high saturation, children's educational
game item icon. Plain white background, single object, centered.
No shadow cast on the background (only the object's own shading). No text, no
logo, no real-world characters. Match the game's world (island explorer theme:
khaki / earth tones, vests, pockets, boots).
```

- 白背景・単体・透過加工前提。各3〜4案生成し、白背景・単体・画風の統一感で選ぶ。
- 帽子/バッジ/メガネ/バンダナ等「体に隠れる部分がない物」は、そのまま単体アイコンで生成可。
- ベルト/リュックのように「装着状態でしか自然に見えない物」は、レイヤーでなく
  **ベース人物ごと画像を切り替える**方式にする（§4・§5参照）。

## 2. ベースキャンバスと整列仕様（★最重要・崩すと全レイヤーがズレる）

基準キャンバス: **1086 × 1448 px**。6体（男女 × back無/キーボード/まほうの本）すべて
この座標に統一済み。新しいベース人物を作るときも**必ず一致**させる:

| 部位 | 位置 |
|---|---|
| 頭頂 | 上から ≈5.9%（≈85px） |
| 足元 | 上から ≈95.6%（≈1385px） |
| 中心 x | ≈49.9%（実質50%） |
| 肩ライン y | ≈580px（≈40.1%） |
| 腰ウエストライン y | ≈905px（≈62.5%） |

ベース人物の生成ルール:
- 正面直立・完全な正面向き・左右対称、腕を体からやや離す、脚を揃える（キャラ選択ポーズ）。
- **体の骨格・ポーズ・各部位の位置は既存ベースと1pxも動かさない。服だけ変える。**
- 帽子はかぶせない（素の頭＝帽子は着せ替えでレイヤー装備）。肩・首・腰周りはすっきり。
- カメラ距離・画角・光源・キャンバス比率は固定。

## 3. スロット構成とレイヤー方式

6スロット。back以外はレイヤー合成（`PlayerAvatar`内で%指定して重ねる）。

- **z順（奥→手前）**: waist → chest → neck → face → head
- **back（リュック）だけは例外**: ベース人物6体の切り替えで表現（正面から隠れる部分が
  大きく、レイヤーでは不自然なため。「隠れる部分を描かない」原則）。
- 同じスロットは1個まで。違うスロットは自由に組み合わせ可。

## 4. 確定アンカー基準（2026-07-08時点・この位置を基本とする）

`top`=上端%（**小さいほど上**）／`left`=中心%／`width`=幅%（いずれもキャンバス基準）。

| アイテム | スロット | top | left | width | 備考 |
|---|---|---|---|---|---|
| コンパスぼうし (head_compass) | head | **1.6** | 50 | 31.3 | b3eで 3.8→1.6（額を出す） |
| はかせぼうし (head_hakase) | head | **1.6** | 50 | 31.3 | コンパスと同一シルエット＝必ず同値 |
| おうかん (head_crown) | head | 4.1 | 50 | 25.8 | ※未実機確認（解放後に要確認・形状別） |
| たんけんメガネ (face_glasses) | face | 16.5 | 50 | 23.9 | 07-07で 20.4→16.5（目の高さへ） |
| たんけんバンダナ (neck_bandana) | neck | 31.8 | 50 | 20.3 | |
| はぐるまバッジ (chest_gear) | chest | 40.5 | 38.7 | 8.5 | 左胸 |
| むしめがねバッジ (chest_magnifier) | chest | 40.5 | 38.7 | 8.5 | 左胸 |

（例: 新しい帽子なら head の 1.6 付近から）。

## 5. 後処理（透過・リサイズ／確立済み）

1. **透過**: 外周フラッドフィルで外側背景を除去 → バウンディングボックス内の
   色距離ベース除去で囲まれた空白（脇・脚のあいだ）を処理。
   **最小blobサイズ閾値 ≈800px** で目の白・靴先の誤消去を防ぐ（実測で調整した値）。
2. **リサイズ**（機械処理・Chat側Python/Pillow）:
   - ベース人物 = 724px ／ 着せ替えレイヤー = 512px ／ どうぐ（バトル消耗品）= 256px
   - 形式は透過PNG（背景等はWebP）。
3. リュックは§3のとおりレイヤーでなくベース人物ごと切り替え。

## 6. アンカー運用（初期値 → 実機微調整の二段構え）

- 初期値 = 基準キャンバスへの実測px を%換算した値。
- 実機で見て `top` を±して上下に追い込む（`left`/`width` は基本不変）。
- 微調整目安（台帳記載の実績値）:
  - 帽子: まだ低い→1.0 前後 ／ 浮いた（額の上に隙間）→2.5 前後
  - ベルト: まだ低い→48 前後 ／ 上げすぎ（お腹に乗る）→51 前後
- 調整したら **必ず deploy まで＋台帳へ記録**（座標微調整も記録する運用）。

## 7. アイテムと学習内容の対応（憲章§4-9「見た目も学びに結びつける」）

コンパスぼうし=じゅんばん／はかせぼうし=パズル全クリア（実績）／おうかん=バッジ全部（実績）／
たんけんメガネ=観察力／たんけんバンダナ=冒険心／はぐるまバッジ=くりかえし／
むしめがねバッジ=じょうけん／どうぐベルト=デバッグ／キーボードリュック=タイピング／
まほうのほんリュック=クイズ。**新モードを足すときは対応アイテムも一緒に設計**する。
