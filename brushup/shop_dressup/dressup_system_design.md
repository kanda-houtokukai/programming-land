# 着せ替えシステム 基盤設計書（案）

作成: 2026-07-07 ／ 対象: 第3波前の準備（まだ案。第3波着手時にCodeへ実装指示として渡す）
目的: 「着せ替えの完成」ではなく「100個以上追加しても設計変更不要な土台」を作る。

---

## 0. 前提（確定済み）

- 着せ替え対象は**プレイヤー(自分)**。相棒は進化するため対象外。
- ベース人物2体（`player_boy_base.png`・`player_girl_base.png`）は完成済み。
  - キャンバス 1086×1448、頭頂=85px(5.9%)、足元=1385px(95.6%)、中心x=49.9% で**男女完全一致**。
  - 正面直立・腕を体からやや離した姿勢・同一光源・透過PNG。
- ゲーム開始時にプレイヤー（男の子/女の子）を選択、以後変更しない。

---

## 1. レイヤー合成方式（結論）

全組み合わせを画像化せず、**部位ごとの透過PNGを決まった順番で重ねる**。
アイテムは「小さく切り出したアイコンPNG＋位置データ(%)」で持つ。これは既存の
ワールドマップ（`AREAS`のleft/top%）・おうちの部屋（家具の%配置）と**同じ考え方**なので、
Codeにとって馴染みのある実装パターンになる。フルキャンバスサイズの絵は不要。

---

## 2. スロット定義とzオーダー（確定案）

| 順 (z) | スロット キー | 表示名 | 内容例 | 同時装備 |
|---|---|---|---|---|
| z1（最背面） | `back` | せなか | リュック・マント | 1つまで |
| z2 | `waist` | こし | ベルト | 1つまで |
| z3 | `chest` | むね | バッジ・ピン | 1つまで |
| z4 | `neck` | くび | リボン・マフラー | 1つまで |
| z5 | `face` | かお | メガネ | 1つまで |
| z6（最前面） | `head` | あたま | 帽子・王冠 | 1つまで |

**ルール**: 同じスロットは1個まで（帽子をかぶったら別の帽子は外れる）。
**違うスロットは自由に組み合わせ可能**（リボン＋帽子＋リュックの同時装備が可能）。
これで「リボンをつけたらリボンしかつけられない」問題を解消する。

### 手に持つ道具について（重要な制約）
ベース人物は**正面直立・腕を自然に下げた固定ポーズ**。虫眼鏡や本を「手で握る」
自然な絵にするには本来ポーズ違いの絵が要るが、それでは「1ポーズ基盤」が崩れる。
そこで、手に持つ系のアイテムは**「腰に留める」「背負う」等、固定ポーズのままでも
違和感のない形に描き直して割り当てる**（下記アイテム選定で対応済み）。

---

## 3. アンカー位置（実測ベースの目安・%座標）

ベース人物（1086×1448・頭頂5.9%・足元95.6%・中心x49.9%）を基準に算出した
初期値。**これは出発点であり、実際のアイテム画像ができてから微調整する**
（ワールドマップの座標を実機で微調整したのと同じ運用）。

| スロット | 目安 top% | 目安 left%(中心) | 備考 |
|---|---|---|---|
| head | 2% | 50% | 頭頂の少し上（帽子のかぶり具合による） |
| face | 15% | 50% | 目の高さ |
| neck | 27% | 50% | 首元 |
| chest | 36% | 50% | 胸の中央 |
| waist | 47% | 50% | 腰のライン |
| back | 36% | 50%（体の後ろ・zで後面に回すので座標はchestと同程度） | 背中 |

各アイテムのアイコンPNGは、この基準点を**中心/上端の目安**として、
アイテムごとの実寸に応じて表示サイズ(width%)を持たせる。

## 4. データ構造（Code向け実装イメージ）

```js
// src/data/dressup-items.js のイメージ
export const DRESSUP_ITEMS = [
  {
    id: "hat_compass",
    slot: "head",           // スロットキー(§2)
    name: "コンパスぼうし",
    skillTie: "junjo",       // 学習内容との対応キー(順次・反復・分岐等)
    image: "hat_compass.png",
    anchor: { top: 2, left: 50, width: 30 }, // %指定・仮値、後で微調整
    acquire: { type: "shop", price: 80 },     // または { type:"achievement", condition:"all_islands_clear" }
    flavorText: "じゅんばんの ちからを みにつけた しるし",
  },
  // ...以降アイテムを追加するだけで済む（コード変更不要）
];
```

- `acquire.type`: `"shop"`（コイン購入）／`"achievement"`（実績解放）／`"default"`（初期所持）
  の3種類をデータで持つ。販売⇔実績の変更は数値/文字列を書き換えるだけで済む。
- `skillTie`: 「そだったちから」の5つの力や学習カテゴリとキーで対応させておくと、
  将来「このアイテムは○○の力を育てた証」という見せ方がしやすい。
- 新アイテム追加 = この配列に1オブジェクト追加＋画像1枚。**コード変更不要**。

---

## 5. 画像仕様（アイテムPNG共通ルール）

- 個別アイコンとして作成（フルキャンバスではない・軽量）。
- 透過PNG、被写体のみでトリミング、余白は少なめで統一（配置時に%widthで揃うように）。
- 画風はベース人物と統一: Pixar調3Dレンダー・柔らかい光源・彩度高め。
- 男女共用前提（体型に依存しない部位＝帽子・バッジ・リボン・ベルト・リュックは
  共用できる。今回選定するアイテムは全てこの「共用可能」カテゴリ）。
