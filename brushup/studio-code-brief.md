# Code に貼る指示文（つくるスタジオ・段階0の起動）

**使い方**: 下の枠内をそのまま Code の新規セッションに貼る。その前に「準備」を済ませておくこと。

---

## 準備（貼る前に神田さんが行う）

1. `studio-assets.zip` を解凍し、中の **22ファイルすべて** を Mac のリポジトリの
   `src/assets/` に置く（card_icon_01〜18・studio-building.png・sign-junbichu.png・
   clapper.png・studio-interior.webp）。
2. 次の4ファイルをリポジトリの `brushup/` に置く。
   - `studio-design.md`（設計の正本）
   - `studio-implementation-stage01.md`（段階0+1の指示書）
   - `studio-map-placement.md`（段階3のマップ設置・確定座標）
   - `studio-block-prototype.html`（**手触りの正本 第11版**）
3. **モデルを Fable 5 に切り替える**（理由は下記）。

---

   合格するまで段階1には進まないでください。

## なぜ Fable 5 か（切り替え提案の3要素）

- **理由**: ドラッグ&磁石スナップの手触り再現と、この後に続く並行実行インタプリタは、
  本案件でこれまでで最高難度の新規エンジン。既存コードの流用がほぼ効かず、長時間の自律実装に
  あたる（dev-workflow §2-2 の「長時間の自律実装」に該当）。
- **コスト面**: Fable 5 は 2026-06-23 以降クレジット制（従量課金）。段階0は範囲を絞ってあるので
  消費は限定的だが、段階1（実行エンジン）はまとまった消費になる見込み。
  ※この提供状況の情報はスキルの変動情報ブロック（最終更新 2026-07-10）に基づくため、
  現在は変わっている可能性がある。
- **選択権**: Opus 4.8 のまま進めることもできる。その場合は段階0を数回に分けて、
  こまめに実機確認を挟む進め方を推奨する。
