## §1. こうぐだな・カードの縮小

### 1-1. 値

| 項目 | 現行 | 変更後 |
|---|---|---|
| こうぐだな幅（`.studio-pal` の `width`） | 24% | **21%** |
| パズルの大きさ（`PAL_S`） | 0.76 | **0.67** |
| カードの よはく（`PAL_GAP_RATIO`） | 0.92 | **0.92（据え置き）** |

### 1-2. 2つを同率で下げることが必須（★形を変えないため）

カードの**幅は `colW`（こうぐだな幅から算出）、高さは `PAL_S`** で決まる。
片方だけ下げると平たく／細長くなり、「形は絶対に変えない」に反する。

**論理幅 `W0 = colW / PAL_S` が一定に保たれること**が、形が変わっていないことの機械的な確認になる。
- 現行（iPad Pro 11 よこ・1194px）: `colW = 127` / `PAL_S = 0.76` → `W0 = 167.1`
- 変更後: `colW ≒ 111` / `PAL_S = 0.67` → `W0 ≒ 165.7`（差 0.8%・許容）

**実装後に実測して `W0` の差が 3% 以内であることを報告すること。** 外れていたら値を調整する。

### 1-3. 下限の根拠（これ以上は下げない）

カード内の文字は幅に追従して自動縮小する（上限16px）。
- 現行: 「ぶつかったら」「もとのばしょ」等の6文字が **14px**
- 変更後: **12px**
- さらに一段（20% / 0.62）下げると **11px** となり、**小1にはつらい**

**12px を下回る値にはしないこと。**

### 1-4. 「ずっと・くりかえし」について

かさばりの主因は**空きスロットの高さ**だが、そこだけ詰めると**形が変わる**ため今回は対象外。
一律12%減で、これらの高さも同じく12%縮む。

### 1-5. 実測の確認（1-2 とあわせて報告）

- 2列が維持されていること（折り返さない）
- カード幅・列間・両端の余白が式どおり
- **studio / gamelab 両方**に適用されていること

## §2. 「おいたよ！」トーストの見切れ

### 2-1. 原因（Chat 調査済み）

`.studio-toast` は `position: absolute; top: 64px`。
**`.studio-wrap` に `position` が無い**ため、基準は `.studio-root`（`position: fixed; inset: 0`）＝**画面の上端**になっている。
ヘッダーの実高は padding 20px ＋ タイトル2段（15px / 11px）で **約56px** ＝ すき間が **8px しかない**。
ヘッダーがわずかに高くなるだけで潜り込む。

さらに **トースト色 `rgba(36,26,44,.92)` とヘッダー色 `#241a2c` が同じ RGB**（36,26,44）なので、重なると溶けて「切れている」ように見える。

### 2-2. 直し方

- **`.studio-wrap` に `position: relative` を付け、トーストの基準をヘッダーの下に移す**。
  `top` は wrap 基準の小さい値（12px 程度）にする。これでヘッダー高が将来変わっても壊れない。
- トーストの DOM 上の親が `.studio-wrap` の外にある場合は、**`.studio-wrap` の中へ移す**。
- ヘッダーと色が同じ問題も直す。**薄い縁を足すか、gl では明るい配色にする**など、重なっても分離して見えるようにする。

### 2-3. ⚠️ 落とし穴

**`.studio-asm` を基準にしないこと。** `.studio-root.big`（ひろげる＝全画面）で `.studio-asm` は非表示になるため、全画面時にトーストが消える。
`.studio-wrap` は全画面でも残る（消えるのは中の子）ので、こちらを基準にする。

**全画面（ひろげる）でもトーストが正しく出ることを必ず確認すること。**

## §6. 参考: この変更が右カラムに与える影響（次の便への申し送り）

こうぐだなが 24% → 21% になると、余った3%は作業エリアとプレビュー側に分配される。
プレビュー（`.theater`）は `aspect-ratio: 3/2` なので、**幅が広がるぶん高さも約12px増える**。

右カラムは iPad で既に約40pxはみ出しており（はいゆうひかえしつが下で切れている）、**この変更で約12px悪化する**。
「ゲームのせってい」の再設計（A-4）で縦予算を組み直す際に、**この12pxを織り込むこと**。
本便では対処しない。
