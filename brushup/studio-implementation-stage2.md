## 2. 保存モデル（★設計の穴を埋める判断・Chat 決定）

設計§7は「draft は1つ・ほぞんで works へ」とだけ定めており、
**「かきかけがあるときに別の作品を開いたらどうなるか」を決めていなかった**。ここを決める。

**規則: 何か（あたらしく つくる／みほん／つくりなおす／コピーして つくる）を開く前に、
現在の draft が「空作品ガード」を通るなら、自動で works に保存してから開く。**

- 空作品ガード = 「動くブロックが1枚以上あるきっかけスタックを持つキャラが1体以上」（設計§8と同一）
- 通らない（＝ほぼ空っぽ）なら黙って捨てる
- → **かきかけは絶対に失われず、確認ダイアログも不要**（設計の「ダイアログ全廃」「空作品ガード」の
  両方と整合する）。ゴミ作品も溜まらない。
- 自動保存された作品の名前は「さくひん{連番}」。

`draft` に `origin` を持たせる: `{ type: "new" | "work" | "sample", id }`
- `ほぞん` を押したとき: origin=work なら**その作品を上書き**、new/sample なら**新規追加**
  （sample のときは `remixOf: サンプルid` を記録）
- draft の形は段階1の `{ bg, sel, chars:[{kind,x,y,stacks}] }` に `origin` を足すだけ

## 3. Home 画面（StudioHome.jsx）

世界観=撮影スタジオ。背景は `src/assets/studio-assets/studio-interior.webp`
（フィルム棚・スポットライト・赤い幕の部屋。中央〜下部が空いているのでUIはそこに置く）。

構成（上から）:
1. **かきかけの さくひん**（draft があるときだけ・タップで続きから）
2. **あたらしく つくる**
3. **フィルムだな**: works を**新しい順**。フィルムのコマ風フレーム＋サムネ＋作品名。
   - タップで4択: **みる** / **つくりなおす** / **コピーして つくる** / **けす**（確認つき）
   - 上限30。超えたら「たなが いっぱい! どれか けしてから」（新規作成をブロック）
4. **みほんのたな**（別枠・削除不可・4本）: 開くと**コピーとして**エディタへ（原本は不変）
   - 初回訪問時のみ、みほんがきらっと光り「みほんを ひらいてみよう」を一言だけ出す
     （押し付けない。フラグは studio 配下に持たず、works.length===0 で判定してよい）

**サムネ（StudioThumb.jsx）**: 画像は保存しない。作品データから背景＋キャラを縮尺配置して
毎回ミニ描画する（設計§7）。論理12×8の縮尺表示＝ステージと同じ式を使う。

作品名: ひらがな8文字まで。未入力は「さくひん{連番}」。名前入力は既存の名前入力UIの作法に
合わせる（5文字制限の実績があるので、そこを参考にしつつ8文字にする）。

## 4. みる（上演専用モード）

**新しいプレイヤーは作らない。** エディタを `showOnly` で開く＝既存の `big`（全画面上演）を
固定し、編集UIを出さない。エンジンも描画も段階1のものをそのまま使う。

- 編集UI（こうぐだな・組み立てエリア・背景セレクタ・控え室・ひろげるボタン）は非表示
- ▶ と ■ は出す（子どもが押す儀式を残す・設計§7「▶で再生され」）
- タップイベントはそのまま効く
- 「とじる」で Home へ戻る
- **showOnly 中は draft を書かない**（人の作品を見ただけでかきかけが壊れないように）

## 5. みほん4本（studio-samples.js）＋ verify

一度に一つの新概念（設計§7）。**下記のデータをそのまま実装する**（内容を創作しない）。

安全なキャラのみ使う（`buildCast` の最低保証＝どの子でも必ず出せる）:
`{type:"player"}` ／ `{type:"mon",id:"mori",stage:1}` ／ `{type:"enemy",id:"slime"}` ／
`{type:"enemy",id:"mushroom"}`

| id | 名前 | bg | 中身 | 盗ませる概念 |
|---|---|---|---|---|
| `dance` | ダンスパーティー | sougen | mori(4,0): はた→**ずっと**{ジャンプ1・おと(ポン)・まわる1}<br>slime(7,0): はた→**ずっと**{まわる1・ジャンプ1・おと(キラン)} | ずっと・**並行**（2体が違うリズムで踊る） |
| `chase` | おいかけっこ | jungle | mori(2,0): はた→みぎへ9<br>mushroom(0,0): はた→**まつ3**→みぎへ9 | **まつ**（時間差）・端で止まる |
| `tap` | タップでへんしん | arena | mori(3,1): はた→ジャンプ1 ／ **タップされたら**→おおきく・おと(キラン)・まわる1<br>slime(8,1): **タップされたら**→きえる・まつ2・でる・おと(ポン) | **タップされたら**・みため |
| `hide` | ドッキリかくれんぼ | daichi | player(0,0): はた→みぎへ6<br>mushroom(6,0): **ぶつかったら**→おと(ドン)・きえる | **ぶつかったら** |

- `tap` の mori に「はた→ジャンプ1」を付けてあるのは、▶を押して何も起きないと子どもが
  戸惑うため（きっかけが tap だけでも `start()` は true を返す実装なので機能上は不要だが、
  最初の合図として置く）。
- おとの `s`: 0=ポン / 1=キラン / 2=ドン。

**`tools/verify-studio.mjs`（新規・verifyチェーンに入れる）** で全みほんを機械検証する:
- bg が BGS に実在 ／ キャラ ≤5 ／ `kindValid` を通る（＝画像が実在する）
- 全ブロックの type が studio-blocks.js に実在
- パラメータが min..max の範囲内
- 入れ子深さ ≤2
- 「ずっと」がスタックの最後にしかない
- 各キャラのきっかけは種別ごとに1本まで／スタックはきっかけで始まる
- 1スタック ≤30ブロック
- **1つでもFAILなら終了コード1**（設計§7「検証FAILは公開しない」）
