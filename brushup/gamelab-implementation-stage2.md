# A. ゲーム機能（中身）

## A-1. うごきカード3種（`DEFS` ＋ `GAMELAB_PALORDER` ＋ `engine.js`）
`gamelab-design.md §5` のレシピに従う。**gamelab 専用**＝`GAMELAB_PALORDER` にのみ追加（studio の `PALORDER` は不変）。`DEFS` は共有カタログに定義追加、アイコンは白チップ内側余白つき（§10）。

- **ランダムにうごく（`moveRand`・うごき青）**：1拍ごとにランダムな方角へ1マス移動（グリッド範囲内にクランプ）。よけ/キャッチの敵に使う。
- **はねかえる（`bounce`・うごき青）**：キャラの現在の向きへ1マス進み、はしに当たったら向きを反転して進む。向きはキャラ状態として保持（初期向きは中央向き or ランダム／design に従う）。
- **相手をゆびさして ぶつかったら（`bumpTarget`・きっかけ橙・hat）**：既存 `bump`（誰とでも）の指定版。**特定キャラ**に重なった時だけ発火（ピルで相手キャラを選ぶ）。キャッチの得点判定に使う。

エンジン実装：`beginBlock`/トリガ判定に上記3分岐を追加。移動は既存のグリッド移動ロジックを再利用。**studio の作品にこれらの型は存在しない＝studio では実行されない**（回帰トレース不変）。単体テストに各カードのケースを追加。

## A-2. ゲームオーバー（ばくだんタッチ）＋ ざんねん画面
段階1で予約した `gameConfig.gameOver` を**中身入り**にする：`gameOver: { targetId }`（`null`＝無効）。
- せってい：ばくだんにするキャラを選ぶ（控室のキャラから1体）。
- 判定（器・`mode.isGame` ガード）：`tickLoop` の毎拍末で、**ばくだん（targetId）が他キャラと重なったら**負け（正確な「誰が触れたら」は `gamelab-design.md §4-5` に従う）。→ 全キャラ停止→**ざんねん画面**（責めない文言「おしい! もういちど?」・紙吹雪なし・「もういちど」「なおす」）。
- クリア判定とゲームオーバー判定の**優先順位**（同拍で両方成立時どちらを出すか）は design に従い固定。

## A-3. じかんクリア＋タイマー＋けっか画面 ／ クリアなし
`clear.type` の "time" と "none" を**ランタイム実装**（枠は段階1で予約ずみ＝スキーマ変更なし・`SCHEMA_VERSION` は上げない）。
- **"time"**：残り時間を拍で計数（1拍=400ms）。タイマーHUD（⏱ 残り）を上演に表示。0で**けっか画面**（スコア発表・中立トーン・「もういちど」「なおす」）。
- **"none"（じゆう）**：自動終了なし（studio的・■でのみ止まる）。
- HUD：段階1のスコアHUDに**タイマー表示**を追加（`clear.type==="time"` の時だけ）。

## A-4. せっていパネル拡張（`WorkshopEditor` ・`mode.isGame`）
段階1のスコアひょうじ＋目標に加えて：
- **クリアじょうけんの種別セレクタ（3択：スコアがたまったら／じかんがおわるまで／クリアなし）**。"score" は目標値、"time" は秒数、"none" は無し、を切替表示。
- **ばくだんタッチ（ゲームオーバー）トグル ＋ ばくだんキャラ選び**。
- 文言はひらがな・低学年でも読める短さ。変更は `scheduleDraft()`。

## A-5. みほん全部（`gamelab-samples.js`）
`gamelab-design.md §5` の全レシピを追加（あつめは段階1で実装ずみ）：
- **よけ**：敵が「ずっと→ランダムにうごく」・ばくだんタッチON・じかんクリア等（design のレシピどおり）。
- **キャッチ**：主人公タップ移動＋「相手をゆびさしてぶつかったら→スコア+1」・スコアクリア。
- みほんのたな＝3レシピ＋あつめ（削除不可枠）。各みほんに `gameConfig` 同梱。

## A-6. 全画面プレイ（ひろげる＝家族に遊ばせる顔）
段階1で HUD/おいわい/けっか/ざんねん は上演に出るが、**「ひろげる」の全画面プレイ体験**を仕上げる：
- 全画面で背景＋キャラ＋HUD＋（クリア/けっか/ざんねん）＋「もういちど」「やめる（編集へ）」。編集UIは隠す。
- `showOnly`（みる＝あそぶ専用）でも同経路で遊べる（draft 不変）。

# B. エディタ見た目作り直し（gamelab のみ・`mode.isGame` ガード）

基準＝`gamelab-editor-mock.html`。**studio は現行の温かい世界観のまま**（studio は `palette-fixes.md` のチップ/スクロール修正だけ）。gamelab の見た目は**全て `mode.isGame` 分岐**で studio 経路に入れない。**凸凹ブロックのパス・エンジンは不変**。

## B-1. こうぐだな（左）：Scratch 寄りに整理（gamelab）
- **枠を廃止**＝カードだけ／**ジャンルは区切り見出しに1回**（各カードの `pname` は出さない）。
- **カードは内容ぴったりの自動幅**（`d.w` 固定でなく、ラベル＋ピル＋余白から算出）＝長い字もピルもはみ出さない。凸凹・チップ（34→30 済）はそのまま。
- **背景は明るい涼しめ**（モック：`#f3f6fa` 系）。
- studio のパレット描画は現行（枠＋pname＋固定幅）のまま＝`mode.isGame` で描画分岐。

## B-2. 組み立てキャンバス（中央）
- **淡いドットのキャンバス**（`radial-gradient` の点・モック参照）＝「スクリプトを組む場所」らしく。
- どのキャラのプログラムか示す `selchip`（丸アイコン＋名前）。
- 既存のブロック配置・ドラッグ・接続ロジックは不変（見た目の地だけ変更）。

## B-3. ステージ＋操作盤（右）
- 上演を**ゲーム画面らしく額装**（角丸＋白フチ）。▶/ひろげるは丸ボタン、スコア/タイマーHUD。
- **せってい・はいけい・控室を「操作盤」パネル**として整える（白カード＋見出し・モック参照）。

## B-4. 全体トーン（私のおすすめ＝合意ずみ）
- 地は**涼しく明るいツール**（studio の茶と別世界）。ただし**角丸・ひらがな・柔らかい色で冷たくしない**＝大人のIDEにしない。
- 色温度・ドット有無・額装の強さは**モックの値を初期値**にし、実機で微調整可。

# C. スキーマ / 検証 / やらないこと

## C-1. スキーマ
- `SCHEMA_VERSION` は**上げない**（`gameConfig` の形は段階1で予約ずみ。`gameOver` に中身、`clear.type` に "time"/"none" を入れるだけ）。
- 旧 gamelab 作品（`gameOver:null`・`clear.type:"score"`）は既定でそのまま動く（フォールバック）。

## C-2. 検証（`npm run verify` 拡張・回帰GREEN）
- エンジン単体：`moveRand`/`bounce`/`bumpTarget` の各ケース、じかんクリアの拍計数、ばくだんタッチの判定を追加。
- `verify-gamelab.mjs`：よけ/キャッチみほんの妥当性（新カード・`gameConfig` の time/gameOver）を追加。
- **回帰ハーネス GREEN**（studio の732イベント・保存モデル・境界スキャン不変）／ベースライン byte 不変（チップ修正の再取得は `palette-fixes.md` 側で完了済みの前提。段階2では geometry を触らない）。
- 実機観測：よけ・キャッチが遊べる／時間切れ→けっか／ばくだん→ざんねん／全画面で家族に渡せる／**studio 完全無変化**。

## C-3. やらないこと（段階3以降・スコープ外）
- **ロジック（もし〜なら・条件・演算・〜までまつ/くりかえす）＝段階3**。**変数・タイマー表示・メッセージ＝段階4**。**マップ開店・教育接続・保護者ガイド・BGM＝段階5**。
- **自分で描いたキャラ（キャラ作成）・自由x/y・リスト・カスタムブロック・ペン・通信＝スコープ外**。
- studio 側の見た目変更（`palette-fixes.md` の2点以外は不変）。

## C-4. 実機ゲート（合格で段階3へ）
iPad＋PC Safari、`#gamelab-dev`：
1. よけ（敵ランダム＋ばくだんタッチ）／キャッチ（相手指定ぶつかり）が**作れて遊べる**。
2. じかんクリア→タイマー→けっか。ばくだんタッチ→ざんねん。全画面プレイで家族に渡せる。
3. せってい：クリア3種の切替・ばくだん選び。保存/かきかけに `gameConfig` 通し。
4. **見た目**：gamelab エディタがモックの方向（涼しく明るいツール／でも温度あり・凸凹そのまま）。**studio は不変**。
5. `npm run verify` 全PASS・回帰GREEN・コンソールエラーゼロ。
