## プロジェクトの基本（前提）
- 神田さんが自分の子ども（小1〜2、将来は高学年も）のために作る、ブラウザ製プログラミング教育ゲーム。
- Vite + React、単一HTML出力、GitHub Pages 公開。公開URL: https://kanda-houtokukai.github.io/programming-land/
- 役割分担: **設計・教育コンテンツ執筆・画像加工＝Chat（この Claude）／ 実装＝Claude Code**。
- モデル運用: 基本 Opus 4.8。大量生成＋検証ループのときだけ Fable 5（終わったら Opus に戻す）。
- 憲章の第一原則: **教育が主・ゲームは動機づけ**。課金なし・通信対戦なし・ガチャなし・実在IP不使用。
- リポジトリの設計正本: project-charter.md / education-curriculum.md / roadmap.md / feature-spec.md
  （＋ progland-handoff.md 台帳、progland-実装指示書.md がリポジトリ側にある）。

## 今日のブラッシュアップで「設計を固めた」もの（9つのメモ・実装待ち）
すべて設計確定済み。実装は Code を触れるとき。各メモの実装指示書/アセット/文章は同梱。

- **メモ01 プロファイル切り替え＆おうちの導線**
  - 起動時のプロファイル選択画面を「島の入口（港）」風に作り直す（要背景生成・未生成）。
  - ヘッダーの名前タップ→おうちを開く（反応しないボタン問題の解決）。
  - おうちは常に「モーダル」で開く（ヘッダー名前・マップの家、両方。閉じたら元の画面）。
  - プロファイル交代は「マップ上の港（起動時選択画面に戻る）」に一本化。おうち内の
    「じぶんをこうたいする」は削除。
- **メモ02 名前の一貫性**（文言のみ・独立）
  - 3階層で役割分担: マップ=短縮名 / ポップアップ=場所名「〜に はいる」 / 中身=ゲーム名。
  - 場所名確定: パズルのしま / クイズのひろば / おえかきのへや / タイピングタワー /
    バトルのアリーナ / おみせ / そだちのもり / おうち。中のゲーム名は変えない
    （将来、同じ場所に別ゲームを足せる余地を残すため）。
- **メモ03 ステージ一覧のUI整理**（難易度別画面全般）
  - 難易度をタブ化（面積圧縮）。難易度から★を外す（色分け＋言葉に。評価の★と混同回避）。
  - 「ホーム」「マップ」の二重ボタンを「◀もどる」1つに（1階層ずつ戻る原則を全画面に）。
- **メモ04 おうちをRPGの部屋に**
  - 部屋の背景（生成済み: room-home.webp）に家具を描き込み、タップで機能へ。
  - 家具割当: 本棚=なかまずかん / 机の日記=きろく / 額縁=プロフィール / 宝箱=将来用 /
    相棒=中央床に別画像で乗せる（進化で変わるため）。
  - タップ表現は控えめ（家具がうっすら光る/揺れる＋押したら反応・世界観優先）。
- **メモ05 ショップをRPGの店に**
  - 店内背景（生成済み: shop-interior.webp、ひげのおじさん店員込み）。
  - 段階1: 店員タップ→「なにを かいにきた?」→「どうぐ/きせかえ」選択。段階2: 今の商品リスト流用。
- **メモ06 コインバランス＆バトルの歯応え**（方向確定・数値は実機で調整）
  - (A) バトルを歯応えあるものに（クイズ難易度をバトルに反映・敵を強く）→ アイテムが要る場面を作る。
  - (C) コイン価格を少し上げる＋導入時一括付与（471枚等）を見直す。
  - 根本問題: 今アイテムを使う必要がない＝バトルが簡単だから。価格だけでは解決しない。
- **メモ07 方眼メモ機能**
  - ロボット命令のクイズ＆**バトル中のロボット命令**に、方眼メモ（指で矢印・番号を書ける）を追加。
  - 普段はたたみ「📝メモ」で開く。トレース（机上デバッグ）・認知負荷の外部化という教育的根拠。
  - メモ前提の難易度にはしない（使いたい子が使う道具）。
- **メモ08 おえかきコードの大改善**
  - A修正: カメが消える（描画領域）・今の色表示・全消し確認・作品名入力。
  - B緩和: ペン上げ下げ命令・スタート位置命令・命令上限緩和。
  - C拡張: 斜め（角度命令・プリセット角度）。※正多角形は高学年拡張で将来。
  - E: ボタンをコンパクト化（方向=十字キー風アイコン・色=ポップアップ・文字を減らす）。
    ※十字キーは「押すと命令が積まれる」挙動（既存方式維持）。
  - 保護者ガイド改訂文あり（art-guide-改訂.js・制限の教育的理由を説明）。
- **メモ09 記録の構造整理＆保護者向け総合案内**
  - ヘッダーの「きろく」削除。子ども向け記録=おうちの日記。
  - 保護者向け「おうちのひとへ」を**マップ最下部**に独立配置（保護者ゲート奥）。
    神田さんの想い「単なるゲームでなく、まず保護者に理念を見てほしい」の表明。
  - 中身（フル版）: 理念文＋各モード教育説明一覧＋記録の見方＋ダッシュボード。
  - 理念文あり（parent-intro理念文.js）。

## 実装の順序（実装ロードマップ.md に詳細）
拡張性を全実装に通底（データは設定ファイル集約・場所とゲーム分離・拡張枠を残す・共通部品化）。
- 第0波: 開発途中（そだったちからF2・ワールドマップF2）を実機確認して地盤固め。
- 第1波: 独立・低リスク（メモ02名前・メモ03ステージUI・ワールドマップF2・そだったちからF2）。
- 第2波: **おうちの塊（メモ01＋04＋09）をまとめて設計・段階実装**（遷移再配線を含む・最重要注意）。
- 第3波: ショップ（メモ05）＋コイン調整（メモ06-C）。
- 第4波: 学びの機能（メモ08おえかき・メモ07方眼メモ・メモ06-Aバトル歯応え）。
- 将来: 高学年拡張（変数・イベント・正多角形）、他ゲームのバトル化、拡張枠の活用。

## 重要な経緯・教訓（次の Claude が知っておくとよい）
- 神田さんのフィードバックは全て「実機で子どもが遊んだ観察」から出る。非常に的確。
- ブラッシュアップの2本の芯: ①メニューを世界にする（ホーム=島・おうち=部屋・ショップ=店）、
  ②教育の本質を守りつつ遊びやすく（制限は理由とともに残し、ただの不便だけ直す）。
- 画像は ChatGPT 生成 → Claude が加工（白背景→透過・トリミング・256px PNG or 1600px WebP）。
  背景は透過不要でリサイズ＋WebP。座標合わせは実機のクロック方向指示が最も正確。
- 「制限をあえて残すなら教育的理由を保護者欄に書く」は神田さんの重要な価値観。
- 大きな改修（おうちの塊・遷移再配線）は段階を切り、段階ごとに実機確認。壊さないこと最優先。
