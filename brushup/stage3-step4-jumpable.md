## §0. ★設計判断: 整数マスのまま実装する（Chat が決めた）

正本 §3 B-1 は「重力＋落下＋着地＋ジャンプを1枚で」とだけ書いており、**座標の扱いを決めていない**。ここを先に決める。

### 0-1. エンジンの現状（Chat が確認済み）

- **キャラの座標は完全に整数マス**。`ch.x += dx`／`ch.y = LROWS - 1`／`Math.floor(...)` で、小数は一切使っていない
- 盤は **12×8 の固定**（`LCOLS=12` / `LROWS=8`）。カメラもスクロールも無い
- **既存の「ジャンプ」カードは演出だけ**。`emitFx(key, {type:"hop"})` を出すのみで、**位置は1ミリも動かない**（CSS の `--hopH` で跳ねて見せている）

### 0-2. 決定

**小数座標を導入しない。整数マスのまま、重力もジャンプも「1拍ないし1ステップで1マス」で表現する。**

理由:
- 小数を入れると `ch.x` / `ch.y` を使う全箇所（当たり判定・端のクランプ・タップ移動・おいかける・ふってくる・ぶつかったら）に波及する。**凍結中の 732イベントを壊すリスクが最も高い道**
- 区切り①で入れた**操作ループ（100ms）と `.actor.op` の等速トランジション**が既にあり、1マスずつの移動でも**なめらかに見えることは実機で確認済み**（b6c 合格）。重力と落下はこれに乗せればよい
- 12×8 の盤で放物線を描いても、マス目が粗いので恩恵が小さい

**重力・落下・ジャンプはすべて操作ループ（`OP_MS`）で進める。** 1拍（400ms）を待たない。

## §1. `jumpable`「とべるように」の仕様

| 項目 | 値 |
|---|---|
| `label`（棚・6文字以内） | **とべるように** |
| `long` | ジャンプできるように |
| カテゴリ | **そうさ**（`dpad` / `tapMove` の次） |
| ピル | なし |
| gamelab 専用 | `GAMELAB_PALORDER` のみ。studio には出さない |

`desc`（ながおしのせつめい）は、**重力がかかることを必ず書く**こと。「とべるように」という名前だけでは「下に落ちるようになる」が伝わらない。
例）「したに おちるように なって、じゅうじキーの ▲で ジャンプ できるよ」

### 1-1. 重力

- このカードを実行したキャラは、以降**操作ループごとに1マス下へ落ちる**
- 何かの上に乗っていれば落ちない

### 1-2. 地面と足場

- **地面＝盤の一番下の行**（`y = LROWS - 1`）。ここで止まる
- **足場＝真下のマスに別のキャラがいること**（同じ `x`・`y + 1`）。その上で止まる
  - 足場になるキャラを限定しない（**どのキャラも足場になる**）。`jumpable` を持つキャラだけが影響を受けるので、既存のみほんは変わらない
- 足場が動いて真下からいなくなったら、また落ち始めること

### 1-3. ジャンプ（じゅうじキーの ▲）

- `jumpable` を持つキャラでは、**▲ は「うえへ移動」ではなく「ジャンプ」**になる
- **地面か足場に接しているときだけ**跳べる（空中で連打しても跳ばない）
- 高さは **2〜3マス**を目安に、上がりきったら重力で落ちてくる。**具体値は Code が決めて、理由とともに報告**すること
- **▼ をどう扱うかも Code が決めて報告**すること（何もしない／速く落ちる、など）
- `jumpable` を持たないキャラの ▲ は**従来どおり「うえへ移動」**

### 1-4. 既存の「ジャンプ」カード（うごき）との関係

うごきカテゴリの既存 `jump`「ジャンプ」は**演出だけで位置が動かない**。`jumpable` と名前が近く、**子どもが混同する恐れがある**。

- **既存の `jump` は変更しない**（studio と共有・実機OK済み）
- **`desc` で違いを言葉にする**こと
  - `jump`（うごき）＝「その場で ぴょんと はねる」
  - `jumpable`（そうさ）＝「したに おちるように なって、▲で とべる」
- ⚠️ **実機で混同するようなら名前を練り直す。** その判断は神田さんの実機確認で行う

## §2. ★「よこスクロール」は実装に存在しない

正本 §3 冒頭（17行目）と §4（98行目）、§6 の⑤は **「横スクロールジャンプ」「よこスクロールみほん」**と書いているが、**このエンジンにカメラもスクロールも無い**（盤は 12×8 固定）。

- **「よこスクロール」は「よこから見た1画面のジャンプゲーム」と読み替える。** 12×8 の1画面に足場を置いて跳ぶ形（初期のアーケードゲームと同じ作り）
- 本当に横スクロールを入れるなら、カメラ・ワールド座標・背景のループが必要で、**凍結中のエンジンとジオメトリの根幹に手が入る**（ベースラインも作り直し）。**本便では扱わない**
- **みほんの追加は本便では行わない。** ジャンプを使ったみほんは、`clone` が揃う**区切り⑤でまとめて**作る

## §5. 検証

- 正本 §5 のとおり、**エンジン単体テストに `jumpable`（落下と着地）のケースを追加**する
