# 設計案: ワールドマップ・ホーム（ホームUIの全面刷新）

作成: 2026-07-05 ／ 状態: 設計案（ユーザーすり合わせ用）
モデル: Opus 4.8（マップ背景はChatGPT生成）

---

## 1. 目的（なぜ変えるか）

現状のホームは、相棒・パズル・クイズ・おえかき・タイピング・バトル・お店・バッジが
**すべて同じ横長カードで上から下に積まれたメニュー画面**。ボタンのリストであって、
「ゲームの世界」ではない。「ちゃんとしたアプリらしくない」。

目指す姿: **ホーム自体が1枚のマップ（村/島）**。そこに「わたしの家」「お店」
「各エリア（ゲーム）」「バトルのアリーナ」が建物やランドマークとして配置され、
子どもは場所を選んで"入っていく"。ボタンのリストではなく、行き先のある世界。
（あつまれどうぶつの森・ポケモンのような「世界に住んでいる」感覚）

## 2. 構造（2階層・確定）

- **ホーム = ワールドマップ**（大きな世界の俯瞰・1枚絵）
- 各エリアはマップ上の建物/ランドマークで、タップで各機能へ入る
- **パズルの島マップは、ワールドマップの1エリア**（「パズルのもり」等に入ると、
  既存のパズル島マップ〔6島・難易度別背景〕が開く）＝入れ子

## 3. マップに配置するもの（ランドマーク）

```
        [ワールドマップ（1枚絵）の配置イメージ]

   ⌨️タイピングの         ⚔️バトルの
      とう                アリーナ(闘技場)
                                    (Lv3で解放・鍵表現)
   🧩パズルの           🏠わたしの家
     もり                (プロフィール/相棒/
   (入ると6島マップ)      ずかん/きろく)

        🎨おえかきの      💡クイズの
          こや            ひろば
                🏪おみせ
              🌱そだったちから
              (家の庭の花壇 or 家の中)
```

配置するランドマークと、タップ後の遷移先:

| ランドマーク | 見た目(案) | タップ後 |
|---|---|---|
| 🏠 わたしの家 | 可愛い家 | 「わたし」情報（プロフィール・相棒・ずかん・きろく への入口をまとめた画面） |
| 🧩 パズルのもり | 森＋ロボット | 既存のパズル島マップ（6島）へ |
| 💡 クイズのひろば | 電球の建物/広場 | クイズ（カテゴリ選択）へ |
| 🎨 おえかきのこや | アトリエ小屋 | おえかきコードへ |
| ⌨️ タイピングのとう | 塔/学校 | タイピングへ |
| ⚔️ バトルのアリーナ | 闘technタジアム | バトルへ（Lv3未満は鍵＋「あいぼうが Lv3で ひらくよ」） |
| 🏪 おみせ | お店（既存のショップアイコン活用） | ショップへ |
| 🌱 そだったちから | 家の庭の花壇 | 「そだったちから」詳細（今作っている5つの花） |

- ヘッダー（上部）: 相棒アイコン＋名前＋⭐＋🪙コインは、マップの上に常時オーバーレイで
  残す（現状のヘッダーを踏襲）。「きろく」「音」ボタンも。
- 「じぶんを こうたいする」（プロファイル切替）は、家の中か設定に置く。

## 4. 見た目の方針

- ワールドマップ背景は**ChatGPTで生成した1枚絵**（村/島の俯瞰）。既存のパズル島マップ
  背景と同じ画風（明るい3Dルック）。→ 世界観が全体で統一される。
- 各ランドマークは、背景に描かれた建物の上に**タップ領域＋ラベル**を重ねる方式。
  （パズル島マップで拠点ボタンを座標配置したのと同じ手法。%座標で管理）
- 未解放（バトル）は建物を少し暗く＋🔒。解放されたら明るく＋通知。
- ひらがなラベルで各場所の名前を表示（低学年向け）。

## 5. パズルの島マップとの関係（既存資産の活用）

- 既存のパズル島マップ（6島・昼夕夜の難易度別背景・拠点座標）は**そのまま維持**。
- ワールドマップの「🧩パズルのもり」→ タップ → 既存パズル島マップが開く、という
  遷移を1本足すだけ。パズル側は改修不要。
- 「ホームに戻る」= ワールドマップに戻る。パズル島マップからは「ワールドへもどる」。

## 6. 遷移の整理（重要）

- 現状「ホーム」に戻る各画面の戻り先を、すべて**ワールドマップ**にする。
- 階層: ワールドマップ →（各エリア）→ 各機能画面。
  - 例: ワールド → パズルのもり → パズル島マップ → ステージ一覧 → プレイ
  - 例: ワールド → クイズのひろば → カテゴリ選択 → プレイ
- 各画面の「◀もどる」「🏠ホーム」の飛び先が正しく繋がるよう配線し直す
  （ここが実装で一番注意が要る＝既存の全モードの導線に関わる）。

## 7. 段階実装（大きな改修なので分ける）

### フェーズ1: ワールドマップの骨格
- マップ背景を敷き、8つのランドマークを配置（タップで各機能へ遷移）。
- ヘッダー（相棒・⭐・🪙・きろく・音）をオーバーレイで維持。
- 各機能への遷移と「戻る」の配線を通す。
- この時点では演出は最小。まず「マップから各所に入って戻れる」動作を完成。
- 【停止して確認】

### フェーズ2: 仕上げ
- 未解放（バトル）の鍵表現・解放通知。
- 「わたしの家」に入ったときのプロフィール/相棒/ずかん/きろくのまとめ画面。
- そだったちから花壇の配置。細かな演出（ランドマークが軽く動く等）。
- レスポンシブ（タブレット縦横）確認。
- デプロイ・実機確認。

## 8. やらないこと / 注意
- パズル島マップの作り直しはしない（そのまま1エリアとして使う）。
- 既存の各機能（パズル・クイズ・タイピング・おえかき・バトル・ショップ・記録・育成）の
  中身は変えない。**変えるのは「入口＝ホーム」と「遷移」だけ**。
- 一度に全部の演出を作り込まない。まず骨格（マップから入って戻れる）を確実に。

## 9. すり合わせたい点
1. マップの世界観（村か、島か、両方の複合か）。既存パズル島マップは「島」なので、
   ワールドは「複数の島がある大きな世界」or「1つの大きな村/町」どちらが良いか。
2. ランドマークの配置（上の図でよいか、家を中央にする等の希望）。
3. 「わたしの家」に何をまとめるか（プロフィール・相棒・ずかん・きろく で良いか）。
4. フェーズ1で一度確認する進め方で良いか。

## 10. 生成する絵
- ワールドマップ背景 1枚（村/町 or 島々の俯瞰・ランドマークを置く余白のある構図）。
  ※プロンプトはすり合わせ後にClaudeが用意。ランドマークの建物自体を背景に
  描き込むか、背景は地形だけにして建物アイコンを別途重ねるか、も設計で決める。
