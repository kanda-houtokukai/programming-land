## §3. ★新カードのラベルを短くする必要がある（b6a のサイズ変更による）
**これが一番重要な補正。**
基準モック `palette-29-structure.html` は **b5x 以前のカードサイズで作られている**。b6a で こうぐだな幅 24%→21%・`PAL_S` 0.76→0.67 と縮めたため、**モックのラベルは棚に入らない**。

### 3-1. 現在の上限
棚のラベルは `fitFont(label, colW − G.LABELX × PAL_S − 8, 16)` で自動縮小する（`G.LABELX = 46`）。
iPad Pro 11 よこ（1194px）での実測・計算:
- `colW = 111` → 使える幅 **72.2px**
- **6文字 → 12.03px**（現在の最小。「ぶつかったら」「もとのばしょ」がこれ）
- 7文字 → 10.3px ／ 8文字 → 9.0px ＝ **小1には読めない**

**棚のラベルは6文字以内**。これが b6a 以降の制約。

### 3-2. モックのラベルは全滅（要短縮）
| モックのラベル | 文字数 | 判定 |
|---|---|---|
| じゅうじキーで うごかす | 11 | ❌ |
| タップした ところへ | 9 | ❌ |
| ジャンプできるように | 10 | ❌ |
| ゴールに ついたら | 8 | ❌ |
| ぶんしんを だす | 7 | ❌ |
| おいかける | 5 | ✅ |
| ふってくる | 5 | ✅ |

### 3-3. 短縮案（`label` を短く・意味は `long` と `desc` が担う）
b5x で導入した3層（`label` / `long` / `desc`）がそのまま効く。**棚では短く、ながおしのせつめいで正確に**。

| 型 | `label`（棚・6文字以内） | `long`（作業エリア/せつめい見出し） |
|---|---|---|
| `dpad` | **じゅうじキー** (6) | じゅうじキーで うごかす |
| `tapMove` | **タップいどう** (6) | タップした ところへ すすむ |
| `goal` | **ゴール** (3) | ゴールに ついたら |
| `chase` | おいかける (5) | おいかける（あいてを えらぶ） |
| `fall` | ふってくる (5) | ふってくる |
| `jumpable`【3-B】 | **とべるように** (6) | ジャンプできるように |
| `clone`【3-B】 | **ぶんしん** (4) | ぶんしんを だす |

⚠️ **`jumpable` の名前に注意**。うごきカテゴリに既存の `jump`「ジャンプ」があるため、`jumpable` を「ジャンプ」に短縮すると**同名2枚**になって子どもが混乱する。上表では「とべるように」としたが、**より良い案があれば提案してよい。ただし6文字以内・既存カードと重複しないことは必須**。

### 3-4. 確認方法
実装後、**gamelab の全29枚について棚での実フォントサイズを測り、12px を下回るものが無いことを報告する**こと。下回るものがあれば、そのラベルを短縮する。

b5x〜b6a はすべて `--update` なしで回帰GREENを保てたが、**段階3は新しいブロック型を7つ足すので、ベースラインは必ず変わる**。

### 手順と合格条件
1. 実装 → 回帰の FAIL 内容を確認
2. `--update` で再取得
3. **diff が以下の条件をすべて満たすことを機械確認する**:
   - `traces` の **732イベントが完全に不変**（新カードは既存みほんで使われないため）
   - `blocks.palorder`（studio 18種）が**不変**
   - 増分が **`blocks.defs` の新7種**と**`geometry.paths` の新しい幅ぶん**だけ
4. 1つでも外れていたら**手を止めて報告**。特に `traces` が動いていたら、studio のエンジン挙動を壊している

**この再取得は段階3で初めてなので、diff の中身を必ず報告すること。**

## §5. じゅうじキーのUIと全画面の関係
指示書 §1 A-1 は「全画面（ひろげる）でも同じく下に表示」としている。
b6a 時点の `.studio-root.big` は以下を `display: none` にしている:
```
.studio-pal / .studio-asm / .bgrow / .castrow / .rcap / .editbtn
```
残るのは **`.studio-wrap` → `.studio-right` → `.theater` と `.gamecfg`**。
したがって **じゅうじキーは `.studio-right`（または `.theater` の直下）に置く**こと。`.studio-asm` や `.castrow` の側に置くと**全画面で消える**。
同じ理由で、b6a の「おいたよ！」表示は `.studio-wrap` 基準に移してある。**新規に画面へ重ねるUIを足すときは、`.big` で消える側に置いていないかを必ず確認する。**
