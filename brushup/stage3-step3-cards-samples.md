## §0. カード一覧の自動生成（★最初にやる・文書のみ）

神田さんが全機能をまとめて実機確認するための一覧を用意する。手書きではなく `DEFS` から生成する。手書きだとカードを足すたびに実装とズレるため。

### 0-1. 素材はすでに揃っている
`src/data/studio-blocks-defs.js` の `DEFS` 28種はすべて `label`（棚の名前）／`long`（正式名）／`desc`（せつめい文）／`cat`（カテゴリ）／`pill`（相手・数・音の指定）を持っている。`PALORDER`（studio 18種）と `GAMELAB_PALORDER`（gamelab 28種）を見れば、どちらの画面に出るかも分かる。

### 0-2. 作るもの
- 生成スクリプト（例: `tools/gen-cards.mjs`）。`npm run cards` で走るようにする
- 出力先は `brushup/cards-reference.md`。⚠️ `docs/` には出さない（公開サイトのビルドに影響させない）
- 内容: 棚の並び順で、カテゴリごとに
  - `label`（棚の名前）／`long`（正式名）
  - どの画面に出るか（つくるスタジオ／ゲームこうぼう／両方）
  - 指定するもの（あいて・かず・おと・なし）
  - `desc`（せつめい文）
- 冒頭に枚数のまとめ（studio ○種／gamelab ○種）を入れる

### 0-3. ズレない仕組み
`npm run verify` に「一覧が `DEFS` と一致しているか」の照合を追加する。食い違ったら FAIL。これで一覧が古いまま deploy されることがなくなる。⚠️ 実装してみて verify が不安定になる・時間がかかりすぎる等の問題があれば、組み込まずに報告すること（その場合は「DEFS を変えたら `npm run cards` を走らせる」を台帳に残す）。

§0 を最初の区切りとして単独で push する（コードの挙動は変わらない）。

## §1. `bump` を gamelab のこうぐだなから外す（28種 → 27種）
正本 §2-1 のとおり。
- `GAMELAB_PALORDER` からのみ `bump` を外す
- ⚠️ `DEFS` からは消さない（studio で使用中）
- ⚠️ studio の `PALORDER` には残す（18種のまま）
- ⚠️ エンジンは引き続き `bump` を解釈する（既存の gamelab 作品が壊れないため）

外したあと、gamelab の「ぶつかったら」は `bumpTarget`（相手指定）1枚になる。同じ名前の2枚が並んでいた状態が解消される。

## §2. カテゴリ見出しの枚数バッジ（★まず判断を報告）
正本 §2-5 は「見出しに枚数バッジ」を求めているが、b5x のこうぐだなUI刷新で意図的に外された可能性がある（現在の実機表示にバッジは無い）。
着手前に `brushup/palette-ui-overhaul.md` を確認し、b5x でバッジを外す判断がされていたかを報告すること。
- 外す判断がされていた → 入れない（b5x で実機OKをもらった見た目を壊さない）
- 特に判断が無かった → 入れてよい。ただし見出しの高さ34px・文字10px・押しやすさを変えないこと
どちらにせよ判断と根拠を報告する。

## §3. みほん3本（正本 §4）

### 3-1. おちものキャッチ
- 主人公＝`じゅうじキー`
- リンゴ役＝`ずっと → ふってくる`
- `ぶつかったら[相手] → スコア ＋`（どちらのキャラに持たせるかは Code 判断・成立することが条件）
- クリア＝じかん（30びょう程度）

### 3-2. おにごっこ
- 主人公＝`じゅうじキー`
- 鬼＝`ずっと → おいかける[主人公]`
- 鬼を ばくだん に指定
- クリア＝じかん（逃げ切り）。※`endGame("timeup")` ＝ 時間まで持ちこたえたらクリア、を確認済み

### 3-3. ★「めいろ」は成立しないので差し替える（Chat 判断）
正本 §4 のめいろは、現状の仕組みでは作れない。
- `gameOver = { targetId: cid }` ＝ ばくだんに指定できるキャラは1体だけ
- キャラ上限は5体。主人公とゴール旗を除くと壁は最大3体だが、触って効くのは1体のみ。残りはすり抜けるので壁にならない

採る案: 壁を1つにして、名前も実態に合わせる。
- みほん名を 「ゴールまで いこう」 にする（「めいろ」と呼ぶと、通路を選んで進むものを期待させてしまい、実物と食い違うため）
- 構成:
  - 主人公＝`じゅうじキー`
  - ゴール旗（動かない）
  - 主人公に `ゴール[旗] → スコア ＋`
  - いわ役を1体置き、ばくだんに指定（よけて進む）
  - クリア＝スコア（1点）
- これで `ゴール` カードの使い方が伝わり、`じゅうじキー` の練習にもなる

採らなかった案と理由:
- ばくだんを複数指定できるようにする → `targetId` を配列に変える＝保存データの形が変わる（スキーマ版7→8）。既存作品の読み込み対応も要る。この便には重すぎる
- めいろのまま壁1つで出す → 名前と実物が食い違い、子どもが混乱する
⚠️ 本格的なめいろが必要になったら、ばくだん複数指定を別便で検討する。本便では扱わない。

### 3-4. みほんの検証
`tools/verify-gamelab.mjs`（または相当の検証）に、3本のみほんが妥当であることを追加する。

- クリア条件と作品の内容が噛み合っている（例: クリア＝スコアなのに得点手段が無い、が起きない）
