## 実装記録（Code・2026-07-12・v2.3-b4b）

### ① ラベルの可読性（RPG定石）
- `.bubble` 地: #FAEEDA→**#FFF9EC**（不透明のまま明度up＝部屋から浮く）。枠 #BA7517 維持。
- **ドロップシャドウ**: `box-shadow: 0 2px 4px rgba(60,40,20,.35)`（＋inset ハイライト維持）＝下地から分離。
- **文字**: 色 #633806→**#59300A**（より濃く）＋`text-shadow: 0 1px 0 rgba(255,255,255,.6), 0 1px 2px rgba(0,0,0,.25)`
  （細い縁取り相当・太い縁は小字で潰れるため控えめ）。
- しっぽ（::before）の色も #FFF9EC に同期。全ラベル共通（おうち/お店。プラザは白チップ式で元から不透明）。

### ② 相棒の進化サイズ（部屋）
- `monsters.js` に共有スケール新設: `partnerStageScale = { 1: 1.0, 2: 1.2, 3: 1.45 }`（初期値）。
- 部屋の相棒: `size = round(92 × scale × sizeK)` → stage1=92 / stage2=110 / stage3=133（k=1時）。
  stage3 の頭がアバター(156)の肩あたり（実測 相棒114 / アバター179 レンダー比 0.64）。
- **足元の床線維持**: `PARTNER_TOP = { 1: 66, 2: 64, 3: 61 }`（大きいほど中心を上げる・b3zと同じ考え方・初期値）。
- mapfloat（生きている感）は据え置き。

### ③ スマホ対応（レスポンシブ縮小）
- 指示の matchMedia(window幅) 案は、Browser paneで `window.innerWidth=0` になる状態があり誤発火した＋
  固定%topとpx sprite の混在は幅でズレるため、**部屋の実幅に比例させる方式に変更**:
  `roomRef`＋ResizeObserver で部屋幅を実測 → `sizeK = clamp(roomW/612, 0.5, 1)`（612=最大幅時の部屋幅）。
  背景と同じ空間でスケールするため**どの幅でも%座標（足元の床線）が崩れない**。デスクトップ/iPadはk=1で不変。
- アバター=`round(156×k)`・相棒=`round(92×scale×k)`＝肩の相対関係を保ったまま縮小。
