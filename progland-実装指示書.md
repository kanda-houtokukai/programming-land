# プログラミングランド v2 — Claude Code 実装指示書

作成日: 2026-07-04（これが正本・v1はChatのアーティファクト）
案件種別: 個人開発 ／ 対象ユーザー: 小学1〜2年生（自宅PC・ブラウザ）

## 0. 前提と移植元

- 同フォルダに置いた `programming-land.jsx` が v1（動作確認済み）。ゲームロジック・デザイン（ステッカー風UI・ひらがなUI・効果音）は原則そのまま移植する。
- 開始手順は dev-workflow スキル §7-3 に従う: ①リポジトリ作成 → ②CLAUDE.md 設置 → ③この指示書を種に `progland-handoff.md` を初期化。**コードはその後**。

## 1. 技術方針（決定事項）

- Vite + React。ビルドは **単一HTMLファイル出力**（vite-plugin-singlefile）。子どもはHTMLをダブルクリックするだけで遊べること。ネット接続・サーバ・アカウント・課金は作らない。
- 保存: `window.storage`（v1のアーティファクト専用API）を **localStorage ラッパーに置き換える**。加えて「きろくの かきだし／よみこみ」（JSONダウンロード・ファイル選択で復元）を保護者画面に必ず実装する（file:// 環境でlocalStorageが消えた場合の保険）。
- データ駆動: ステージ・クイズ・タイピング課題はすべてデータファイル（ts/json）に分離。本体コードに問題をハードコードしない。

## 2. 難易度システム（全モード共通・決定事項）

- 3段階: ⭐やさしい ／ ⭐⭐ふつう ／ ⭐⭐⭐むずかしい。モード入口で選択、いつでも変更可。
- 記録は難易度別に保存・集計（保護者画面に難易度別の到達度を表示）。
- 解放条件は難易度内で完結させる（むずかしいを遊ぶのに、やさしいの全クリアは要求しない）。

## 3. フェーズ計画と完了条件

### P0: 移行（v1同等の完成）
- v1の3モード＋きろく画面が単一HTMLで動く。localStorage保存・書き出し/読み込みが動く。
- 完了条件: v1の全機能が動作し、既存15ステージがソルバー検証（後述）を通る。

### P1: パズル大増量（150ステージ以上）
- ワールド6つに拡張: ①じゅんばん ②くりかえし ③もしも ④くみあわせ（②＋③混合）⑤あたまのたいそう（最短ブロック競争）⑥ちょうせんじょう（大型迷路）。
- 各ワールド×3難易度×8〜10ステージ ≒ 150〜180。
- **生成スクリプト＋自動検証を必ず作る**: ステージはジェネレータで量産し、ソルバーで「★3解が存在」「★3のpar値が最短解と一致」「意図した難易度レンジ（解の長さ・使用ブロック種）」を全数チェック。検証を通らないステージは採用しない。手作業で150面作らないこと。
- 完了条件: `npm run verify` で全ステージ PASS がCI的に確認できる。

### P2: クイズ大増量（200問以上）
- 既存4カテゴリ＋「よみとり（フローチャートの絵を読む）」を追加し5カテゴリ×3難易度。
- きまり発見・ロボット命令系は**テンプレート生成**（絵文字パターンや歩数を機械生成し正答も機械算出）で量産可。じゅんばん・なかまわけ系は固定問題を書き起こす。誤答が成立しない問題（正解が複数ある等）を防ぐ生成ルールを設ける。
- 完了条件: 総問題数200以上、各問題に解説文、難易度タグ付与、出題はセット内シャッフル。

### P3: タイピングモード（新規）
- 小1〜2向け3段階:
  1. やさしい「キーを さがせ」— 画面のキーボード図で光ったキーを押す（ホームポジション導入、指の色分け）
  2. ふつう「ひらがな ことば」— 2〜4文字のことばをローマ字入力（画面に押すキーのガイド常時表示）
  3. むずかしい「ぶんしょう」— 短文、ガイドは薄く
- ローマ字は訓令式・ヘボン式の両方を正解にする（si/shi, ti/chi 等）。**変換テーブルの単体テストを書く**。
- 記録: レベル別ベスト（正確率・1分あたり文字数）、練習した日を既存ログに合流。
- 完了条件: 外付けキーボードで全レベルが遊べ、テスト PASS。

### P4: 記録・バッジ拡張
- バッジを難易度・タイピング対応で25個程度に増補。保護者画面に難易度別到達度と「キーボード入力」項目を追加。
- 完了条件: 書き出したJSONを読み込むと完全復元できる。

## 4. やらないこと
- オンラインランキング・通信機能・広告・外部アカウント連携
- v1のUIトーン（ひらがな・ステッカー風・効果音）の全面刷新
- 実在キャラクター素材の使用（絵文字と自作SVGのみ）

## 5. 検証と落とし穴
- 各フェーズ完了時に必ずビルド→ブラウザ実機（Safari含む）で確認。プレビュー成功≠完成。
- dev サーバ稼働中に `npm run build` を実行しない（.next/ビルド破損の既知事故と同型）。
- 効果音は AudioContext をユーザー操作後に初期化（自動再生制限対策。v1実装を踏襲）。
- 保存データはバージョン番号を持たせ、将来の項目追加でクラッシュしない読み込み（デフォルト値マージ）にする。v1実装の `{...newSave(), ...parsed}` 方式を踏襲。

P0完了時・P1のジェネレータ＋ソルバー完成時（量産前）・各フェーズ完了時に一度停止して報告。
