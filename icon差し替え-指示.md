# 追補: 絵文字 → オリジナル画像への差し替え

作成: 2026-07-04 ／ 親: progland-実装指示書.md
対象: ゲーム内メインキャラ（パズルのロボット・おえかきのカメ）＋ホームの3モードアイコン

## 目的
これまで各所で使っていた絵文字（🤖🐢🏠💡🎨等・OS依存で見た目が変わる借り物）を、
オリジナル画像に差し替えて統一感と「自作教材らしさ」を出す。特にゲーム中のメイン絵
（パズルのロボット、おえかきのカメ）を優先する。

## 素材（すべて透過PNG・256px正方形・軽量化済み）
`src/assets/` に追加する:
- `robot.png` … ドット絵ロボット（正面）。パズル盤面のプレイヤー。
- `turtle.png` … ドット絵カメ（右向き）。おえかきコードのカメ。
- `icon_puzzle.png` … 3Dルックのロボット。ホームの「ロボットパズル」モードアイコン。
- `icon_quiz.png` … 3Dルックの電球。ホームの「かんがえるクイズ」モードアイコン。
- `icon_art.png` … 3Dルックのカメ。ホームの「おえかきコード」モードアイコン。

※ robot/turtle は**ドット絵**（NEAREST拡大でカクッと表示・pixelated指定）。
※ icon_* は**3Dルック**（通常の滑らか表示）。用途が違うので混同しない。

## 実装

### 1. パズル盤面のロボット（最重要）
- 現在絵文字 🤖 で描いているプレイヤーを `robot.png` に差し替え。
- **4方向対応**: robot.png は正面1枚。盤面で向き（右/下/左/上）を表現するには
  CSS transform の rotate で回すのが最も手軽で絵柄が完全に揃う。
  例: 右向き=rotate(0)〜という単純割り当てでもよいし、
  「正面＝下向き基準」にして down=0°, left=−90°回転…のように統一する。
  ※ 厳密な「横顔」を求めず、1枚を回して4方向にするのが実装も見た目も安定。
- 盤面1マスに収まるサイズ（既存の絵文字と同じ表示枠）。`image-rendering: pixelated`。
- 移動アニメ（マス間の滑らかな移動）は既存の仕組みをそのまま流用（画像に差し替わるだけ）。

### 2. おえかきコードのカメ
- 現在絵文字 🐢 で描いているカメを `turtle.png` に差し替え。
- カメは**右向き**素材。進行方向に応じて左右反転（scaleX(-1)）で左右を表現。
  上下向きが必要なら rotate で対応（おえかきは軌跡が主役なのでカメの厳密な向きは簡易でよい）。
- ギャラリーのサムネイルはカメ非表示のまま（軌跡だけ）。プレイ中のみカメ表示。

### 3. ホームの3モードアイコン
- ホーム画面のモード選択に出している絵文字（🤖💡🎨）を、対応する icon_*.png に差し替え。
  ロボットパズル=icon_puzzle / かんがえるクイズ=icon_quiz / おえかきコード=icon_art。
- こちらは3Dルックなので通常表示（pixelated 不要）。角丸カードの中に収める既存レイアウト流用。

## やらないこと（今回のスコープ外）
- 🔒🔊✅⭐🏠📖 等の**機能アイコンは絵文字のまま**（差し替えの労力に見合わない・実用上問題なし）。
- 相棒モンスター（既存の自作SVG）は変更しない。
- 島マップ拠点アイコンの画像化は今回やらない（将来必要なら別途）。

## 検証観点
- パズル: ロボットが盤面で正しく表示され、4方向の向きが分かる。移動アニメが破綻しない。
  ドット絵がにじまず、iPad Safari でくっきり出る（image-rendering 効いているか）。
- おえかき: カメが表示され、進む向きに応じて左右が変わる。軌跡描画は従来どおり。
- ホーム: 3モードアイコンが差し替わり、3Dルックがきれいに出る。
- ビルドサイズが極端に増えない（5点合計 約320KB・問題ない想定）。
- `npm run verify` は従来どおり全PASS（画像差し替えはロジックに無関係）。

## 備考
- robot/turtle はドット絵、icon_* は3Dルックという「2つの画風の混在」は意図的。
  ゲーム中＝ドット絵（プログラミングらしさ）、看板＝3Dルック（親しみやすさ）の役割分担。
  将来どちらかに統一したくなったら、この方針から見直す。
- ロボットの「横顔」を別途用意したくなった場合は、正面回転では不自然になるので、
  その時に専用素材（ドット絵で左右対称に揃えたもの）を追加する。今は回転で足りる。
